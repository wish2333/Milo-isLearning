# M7.8 Implementation Plan — Pre-Launch Optimization

> **For agentic workers:** REQUIRED SUB-SKILL: Use `subagent-driven-development` or `executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 M7.6 结项后的产品从"核心闭环可走通"推进到"可上线发布"。覆盖 PRD-Report §7.1 的 11 项上线阻塞缺口（B1–B11）和 §8 Phase 1–2 路线图：可观测性基础（埋点 / 评分 / Error Boundary）、快速体验优化（字体平衡 / 导航模块化 / 首页智能路由 / 蒙对标注）、学习工具（错题本导出 / 重刷错题）、间隔重复系统（跨概念错题重温）。

**Architecture:** M7.8 继续保持 local-first、BYOK、零后端。所有新增数据字段（`guessed` / `reviewSlots` / 评分 / 埋点）均为 optional，旧 Module 可继续学习和导入。埋点数据存 LocalStorage，批量上报为可选云端扩展。间隔重复不改变状态机语义，只在 `advance()` 的 concept 切换分支注入复习题。

**Tech Stack:** Next.js App Router, React 19, TypeScript strict, Zustand persist, LocalStorage repository, Zod, Vitest, Playwright, Tailwind CSS, `next/font`。

**PRD 依据：** `docs/v1.0.0/PRD.md` V2.0 — FR-09~FR-12、§9.5 顺序流契约、§9.6 间隔重复算法、§2.4 字体 Token、§10.6 三层验证架构、§14 上线路线图。审计依据：`docs/v1.0.0/PRD-Report.md` §6 NP-08~NP-14、§7.1 B1–B11、§8 Phase 1–2。

## Global Constraints

- 不引入服务端持久化，不增加账号系统，不做云同步。埋点数据默认存 LocalStorage，云端上报为可选扩展且不阻塞核心流程。
- 不把 API Key 写入导出文件、埋点上报、错题本 Markdown、URL 或日志。
- 所有新增数据字段必须是 optional；M7.6 及更早编译的 Module 必须可继续学习、导入、导出。
- `guessed`、`reviewSlots`、评分等字段缺失时必须优雅降级，不得阻塞渲染或判分。
- 顺序学习流契约（PRD §9.5）：间隔重复的复习题只能作为额外 slot **插入**，不替换原有题号、不改变 `quizIndex` 语义。
- 字体平衡只改 CSS 变量值，不改组件类名；字体加载通过 `next/font`（不引入远程 `@font-face` 阻塞首屏）。
- 全局导航组件化不得破坏现有 `/learn/module/[id]` 的 LearnShell 嵌套。
- 错题本导出和重刷页面不得干扰主进度（`attemptVersion` 递增但 `computeMastery` 忽略 >0 的版本）。
- 所有删除 / 清理操作保持二次确认。
- 不收紧 M7.6 的 LLM Schema；M7.8 不改编译期 Prompt。

## 0. 审查修订摘要

M7.6 完成了题目教学质量、本地判分、顺序学习流、UI design 落地。PRD-Report（V1.2）审计结论：核心闭环符合度 95%，但存在 **11 项上线阻塞缺口**（B1–B11）和 **7 项新产品决策**（NP-08~NP-14）尚未实现。PRD 已据此修订为 V2.0（新增 FR-09~FR-12）。

M7.8 将 PRD-Report §8 的 Phase 1（可观测性基础）和 Phase 2（产品体验优化）合并为一个里程碑，拆分为 4 条工程主线：

1. **Stream A — 可观测性与上线基础**：埋点系统、完成页评分、全局 Error Boundary。解决"无法衡量北极星指标"问题（B1/B2 + S5）。
2. **Stream B — 快速优化**：字体平衡与加载、全局导航模块化、首页智能路由、蒙对自报标注。对应 NP-08/NP-13/NP-14/NP-09（B5/B10/B11/B6）。
3. **Stream C — 学习工具**：错题本 Markdown 导出、重刷错题独立页面。对应 NP-11/NP-12（B8/B9）。
4. **Stream D — 间隔重复**：跨概念错题重温系统。对应 NP-10（B7）。

设计取舍：

- M7.8 不做真正的服务端埋点管道；LocalStorage 批量存储 + 可选 `navigator.sendBeacon` 上报，云端接收端推迟到 M8。
- M7.8 不做断点续编（True Stage Resume）；M7.5 的 Compile Recovery V1 已够用，推迟到 V1.1。
- M7.8 不做移动端系统测试 / 跨浏览器测试 / a11y 审计（S1/S2/S3）；这些是验证类工作，放在内测阶段。

**机制（PRD §9.6）：**

```
Concept N 结束时（advance 到 conceptIndex+1）：
  1. 收集 Concept N 中 score<80 或 guessed=true 的 slot → 注入 Concept N+1 reviewSlots
  2. 收集 Concept N-1 中首次做对的 slot → 注入 Concept N+1 reviewSlots（确认掌握）

Concept N+1 渲染时：
  1. 先按原顺序做新题（quizIndex 0..M-1）
  2. 新题耗尽后，呈现 reviewSlots 中的复习题（标注"复习"）
  3. 复习题答对 → 不再重现；答错 → 继续携带到 N+2
```

- [ ] **Step 2: advance() 注入 reviewQueue**

```ts
case 'concept': {
  // ...现有 quizIndex + 1 逻辑...

  // 跳到下一 Concept 前，收集当前概念的错题
  if (conceptIndex + 1 < currentModule.concepts.length) {
    const wrongSlots = collectReviewSlots(currentModule, conceptIndex, attemptsBySlot)
    const confirmSlots = collectConfirmSlots(currentModule, conceptIndex - 1, attemptsBySlot)
    const reviewSlots = [...wrongSlots, ...confirmSlots]
    set({
      stage: {
        kind: 'concept',
        conceptIndex: conceptIndex + 1,
        quizIndex: 0,

  }
```

| 间隔重复复杂度高，可能引入状态机 bug | 学习流卡住或跳题 | `reviewSlots` 为 optional；旧 Module 无此字段时按空数组；Playwright 覆盖概念切换 E2E |
| `next/font` 加载 Fraunces 增加首屏体积 | FCP 退化 | `display: 'swap'` + `subsets: ['latin']`；仅加载 4 个 weight；中文不受影响 |
| 全局 layout.tsx 与 LearnShell 双重导航 | `/learn/module/[id]` 出现两个 nav | LearnShell 内部已渲染自己的 nav；确认 layout.tsx 不与 LearnShell 冲突；E2E 覆盖 |
| 蒙对按钮滥用（用户全标蒙对） | 掌握度数据失真 | 按钮低调灰色；教育产品鼓励诚实；内测阶段统计 `guessed/score=100` 比值监控 |
| 埋点 LocalStorage 占满配额 | 其他数据写入失败 | 批量缓冲有上限；quota 异常时丢弃最旧事件；不阻塞核心流程 |
| 首页改 client component 失去 SSG | SEO / 首屏性能 | 首页内容简单，SSR 初始渲染用默认值；hydrate 后更新；影响可忽略 |
| 错题本 Markdown 格式在特殊字符下错乱 | 导出文件不可读 | stem/answer 中的 Markdown 特殊字符转义；测试覆盖含特殊字符 fixture |
