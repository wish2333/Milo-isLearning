## 0. 结论

M7.8 已完成，并在最终验证中通过。项目从"能按预期顺序学习"进一步推进到"可上线运行"——补齐了上线前 11 项阻塞缺口，将埋点、评分、Error Boundary、字体平衡、全局导航、智能路由、蒙对标注、错题本导出、重刷错题、间隔重复全部落地。

这轮最关键的变化是四类能力缺口被收住了：

1. **可观测性从零到一**：`analytics.ts` 轻量埋点系统建立，10 个最小事件覆盖编译到完成全链路；完成页 5 星评分和全局 Error Boundary 补齐了 UX 反馈闭环。

4. **向后兼容零破坏**：所有新增字段（`guessed`、`reviewSlots`、评分、埋点）均为 optional；M7.6 及更早 Module 可继续学习、导入、导出，无任何降级。

**核心成果**：

## 2. 架构决策

| 决策 | 选项 | 选择 | 原因 |
|------|------|------|------|
| 埋点存储 | IndexedDB / LocalStorage / sendBeacon | LocalStorage + sendBeacon | 零后端依赖；内测阶段手动导出足够；M8 再接云端 |
| 间隔重复实现 | 修改 quizIndex 语义 / 独立 reviewIndex | quizIndex 自然延伸到 `M + reviewSlots.length` | 保持 `advance()` 单索引推进；ConceptView 本地态判断是否复习题 |
| 蒙对数据模型 | `guessed` 独立表 / AttemptRecord 字段 | `AttemptRecord.guessed?: boolean` | optional 字段，零迁移成本 |
| 全局导航 | 每个页面单独引入 / layout 统配 | `learn/layout.tsx` + `usePathname()` 抑制 module/[id] 双重导航 | 最小改动，不破坏 LearnShell 结构 |
| 重刷错题存储 | persist / 内存级 | 非持久化 Zustand | 刷新即丢失符合预期；不干扰主进度 |

## 4. 已知限制

1. **中文未走 next/font**：思源宋体体积过大，M7.8 只加载 Fraunces（拉丁衬线）。中文走系统回退（Songti SC / SimSun），M8 用 `cn-font-split` 分包加载。
2. **埋点无云端端点**：`ENDPOINT = undefined`，事件只写 LocalStorage。内测阶段手动导出，M8 接 `/api/events`。
3. **间隔重复无配置开关**：「确认掌握题」（Concept N-1 正确题在 N+1 重现）M7.8 固定开启。内测观察是否打扰，M8 可加配置 toggle。
4. **重刷页不计入学习时长**：review 测试写入 attempts-store（attemptVersion > 0），不影响 `computeMastery`。学习时长统计 M8 再议。
5. **断点续编未实现**：M7.5 Compile Recovery V1 已够用，True Stage Resume 推迟到 V1.1。
6. **Playwright E2E 未更新**：M7.8 约定不跑 e2e，新增功能的回归覆盖延后到内测阶段。

## 5. 风险登记回顾

| 风险 | 实际表现 |
|------|---------|
| 间隔重复引入状态机 bug | 未出现。`reviewSlots` optional，旧 Module 空数组降级；advance() 逻辑在单测覆盖 |

| 首页改 client component 失去 SSG | 首页内容简单，SSR 初始渲染默认值，hydrate 后更新，影响可忽略 |
| 错题本 Markdown 特殊字符错乱 | `escapeMarkdown()` 函数对 `*_`#[]` 转义，测试覆盖含特殊字符 fixture |
