## 0. 结论

V1.0.0 MVP Final Polish 已完成，并在最终验证中通过。项目从"可上线运行"推进到"上线体验完善"——补齐了 4 项审计推荐功能，将展示模式 LLM 反馈、SEO 分享预览、蒙对标注撤销、间隔重复配置开关全部落地。

这轮最关键的变化是四类体验缺口的收口：

1. **展示模式 LLM 反馈可用**：`/api/feedback` 和 `/api/feynman-eval` 在客户端无 BYOK 配置时，fallback 到服务端环境变量。展示模式下的 FillBlank 语义兜底和费曼评分不再 400 报错。
2. **SEO 分享体验补齐**：`layout.tsx` metadata 扩展 OpenGraph / Twitter Card / icons / robots 字段，分享链接时生成预览卡片。
3. **蒙对标注双向可逆**：`attempts-store` 新增 `unmarkGuessed` action，FeedbackPanel 标注后显示"撤销"链接。用户误标注后可一键回退，减少操作焦虑。
4. **间隔重复用户可控**：`settings-store` 新增 `confirmReviewEnabled` toggle（默认开启），用户可在 Settings 页关闭"确认掌握题"注入。

**核心成果**：

1. API env fallback helper（`getEnvLLMConfig()` + 3 场景单测）
2. SEO metadata 扩展（openGraph + twitter + icons + robots）
3. 蒙对标注撤销（`unmarkGuessed` action + FeedbackPanel "撤销"链接 + 4 项往返单测）
4. 间隔重复配置开关（`confirmReviewEnabled` 字段 + progress-store 条件调用 + ProductionSettings toggle + 3 项单测）
5. `bun run typecheck`、`bun run lint`、`bun run test`、`bun run build` 全部通过（341 tests）

## 2. 架构决策

| 决策 | 选项 | 选择 | 原因 |
|------|------|------|------|
| env fallback 实现位置 | 路由内联 / 共享 helper | 共享 helper `env-fallback.ts` | 两个路由复用同一逻辑；DRY；单测可独立验证 |
| env fallback 触发条件 | 总是使用 env / 仅 llmConfig=null 时 | `llmConfig ?? getEnvLLMConfig()` | BYOK 优先（实用模式不受影响）；展示模式自动 fallback |
| `unmarkGuessed` 字段移除方式 | 设为 `false` / 解构移除 | 解构移除（`{ guessed: _, ...rest }`） | `guessed === undefined` 与"从未标注"语义一致；`computeMastery` 检查 `=== true` 不受影响 |
| `confirmReviewEnabled` 持久化 | 新增 storage key / 复用 `alc:settings` | 复用 `alc:settings` | 同属用户设置；Zustand persist 自动处理旧数据缺失字段（默认 `true`） |
| progress-store 读取 settings | React hook / `getState()` | `useSettingsStore.getState()` | 非 React 上下文调用；避免循环依赖 |
| SDD 工作流 | 手动实施 / subagent 驱动 | Subagent-Driven Development | 4 个 Task 独立可并行设计；每 Task implementer + reviewer 双重把关；controller 负责进程和最终验证 |

## 4. 已知限制

1. **`openai-compat` 默认 baseURL 为空字符串**：`env-fallback.ts` 中 `PROVIDER_DEFAULT_BASE_URL['openai-compat']` 为 `''`。展示模式默认使用 DeepSeek，`openai-compat` 需用户在 `.env.local` 中设置 `OPENAI_COMPAT_BASE_URL`。V1.1 可补充运行时检测和提示。
2. **蒙对/撤销按钮缺少 `aria-label`**：FeedbackPanel 中"其实是蒙对的"和"撤销"按钮均无 `aria-label`，与现有按钮风格一致。V1.1 统一做 a11y 审计时补充。
3. **`clear()` 不重置 `confirmReviewEnabled`**：`settings-store.clear()` 仅清除 `config`（LLM 配置），不重置学习偏好。设计决策：clearing LLM config 不应重置学习偏好。如需独立重置，V1.1 可加 `resetPreferences()` action。
4. **R2/R3 浏览器交互验证未执行**：R2（蒙对撤销 UI 流）和 R3（toggle 交互）的 state 逻辑由单测覆盖，UI 接线由 code review 验证，但未在真实浏览器中点击验证。内测阶段补充 Playwright E2E。
5. **R4 env fallback 端到端未验证**：需要配置真实 `DEEPSEEK_API_KEY` 并在展示模式下触发 FillBlank 语义兜底。helper 逻辑由 3 项单测覆盖。

## 5. 向后兼容与迁移

### 零迁移成本

所有新增数据字段均为 optional，旧数据可继续使用：

| 字段 | 位置 | 缺失时默认值 | 处理方式 |
|------|------|------------|---------|
| `confirmReviewEnabled` | `settings-store` (LocalStorage `alc:settings`) | `true` | Zustand persist 反序列化时使用初始值 |
| `onUnmarkGuessed` | `FeedbackPanel` props | `undefined` | 条件渲染守卫：`isGuessed && onUnmarkGuessed` |
| `unmarkGuessed` | `attempts-store` action | — | 新增 action，不影响现有 store 接口 |

### 不变式保持

- `computeMastery` 仍然检查 `guessed === true`（非 truthy）；`unmarkGuessed` 产生 `undefined`，掌握度恢复计入
- `collectConfirmSlots` 函数本身不变；`progress-store` 仅条件调用，`confirmReviewEnabled=true` 时行为与 M7.8 完全一致
- 状态机转移规则未修改；`advance()` 的 concept 切换分支仅增加 `confirmReviewEnabled` 条件判断
- API env fallback 仅在 `llmConfig === null/undefined` 时触发；客户端携带 config 时走正常 BYOK 路径
