# M8.1 Implementation Plan — 主题归类与主题刷题

> **For agentic workers:** REQUIRED SUB-SKILL: Use `subagent-driven-development` or `executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为题库（Module）增加「主题（Topic）」归类维度。用户默认仍走当前的分概念学习流程；但在题库管理中可将多个题库归入一个主题，进入「主题刷题」模式——按可调整的顺序依次学完主题下的每个题库（完整 8 阶段流程），模块间以过渡页衔接。同时扩展错题/蒙对题重刷机制：新增独立的「模糊题（蒙对题）重刷」模式，主题模式下支持跨题库错题聚合重刷。

**Architecture:** M8.1 保持 local-first、零后端。新增 `Topic` 领域模型（一对多：一个题库最多归属一个主题），持久化到 LocalStorage（`alc:topic-index` + `alc:topic:{id}`）。新增 `useTopicSessionStore`（持久化）编排主题内多模块顺序，在 `module/[id]/page.tsx` 的 done 阶段重定向处拦截——若有活跃主题会话则跳转主题过渡页而非 `/learn/done`。每个题库仍走完整的现有 `ModuleStage` 状态机（不做修改），主题层只在模块间编排。review-store 扩展为支持多 moduleId 聚合 + reviewFilter 筛选（`all | wrong | guessed`）。

**Tech Stack:** Next.js App Router, React 19, TypeScript strict, Zustand, 无新增依赖。

**依据：** `docs/v0.1.0/Technical-Specification.md` §5 Runtime / §6.2 Store split；`docs/v0.1.0/PRD-Report.md` NP-10（跨概念间隔重复）/ NP-11（错题本导出）/ NP-12（独立复习页）；`src/types/domain.ts`（Module / ModuleStage / AttemptRecord / Mastery）；`src/lib/state/review-store.ts`（现有错题重刷）；`src/lib/runtime/adaptive-sequencer.ts`（collectReviewSlots / collectConfirmSlots）。

**已确认的产品决策（2026-07-11）：**

| 决策点 | 选择 |
|--------|------|
| 主题与题库的关系 | **一对多** — 一个题库最多归入一个主题；未归入的题库 `topicId` 为空，走默认流程 |
| 模块间过渡 | **过渡页** — 「题库A 已完成，下一个：题库B」，展示主题进度 (2/5)，用户点击「继续」 |
| 单库流程范围 | **完整 8 阶段流程** — 每个题库走 module_intro → concept → challenge → feynman → done，主题层只编排顺序 |
| 模糊题重刷 | **独立模式** — 新增「只刷蒙对题」入口与「重刷错题」并列；review 页加筛选 tab（全部 / 仅错题 / 仅蒙对） |
| 主题错题范围 | **跨题库聚合** — 主题模式「重刷错题」聚合主题下所有题库的错题+蒙对题，打乱混合出题 |

## Global Constraints

- **不修改 `ModuleStage` 状态机。** 7-state discriminated union（module_intro → concept → challenge → feynman_* → done）保持不变。主题层在状态机**之上**编排，不侵入单库学习流程。
- **不改变默认学习入口。** 用户从题库卡片直接「开始学习」仍走当前的分概念单库流程（无主题）。主题刷题是从题库管理中的主题卡片**显式进入**。
- **一对多归属。** 一个题库最多归入一个主题。将题库加入主题B时自动从主题A移除（UI 层保证）。删除主题时题库变为未归属（不删除题库本身）。
- **题库 ID 稳定性。** 主题通过 `moduleIds: string[]` 引用题库 ID。题库被删除时，主题中的引用同步移除（级联清理）。
- **主题可通过 `.alc-topic.json` 导出/导入。** 导出包含 topic 元数据 + 有序的完整模块包数组（`CompiledModulePackage[]`）。导入时重新分配所有 ID + 创建本地 Topic。安全规范与 `.alc-module.json` 一致（拒绝 apiKey）。`CompiledModulePackage` 自身格式不变。
- **主题功能在展示模式和实用模式下均可用。** 主题创建/编辑/导出/导入/刷题/错题重刷均为纯本地操作（无 LLM），两种模式行为一致。展示模式额外支持从 `manifest.json` v2 加载预配置主题作为 demo 数据（Must 基础设施，Should 示例内容）。
- **不影响 mastery 计算。** `computeMastery` 是 per-module 的纯函数，不感知主题。主题完成度由各模块 mastery 聚合得出，不修改 mastery.ts。
- **review-store 保持非持久化。** 主题刷题的跨库错题聚合也使用 volatile review-store，刷新即丢失（与现有单库 review 行为一致）。
- **PRD 不新增 FR 编号。** 主题归类是对 NP-10~NP-14 间隔重复/错题体系的自然扩展，不新增独立 FR。

## 0. 背景与动机

### 0.1 现状

M7.8~M8 完成后，产品的学习闭环已经完整：

- **单库学习流程**：导入 Markdown → 8 阶段编译 → module_intro → concept（含自适应 reviewSlots 注入）→ challenge → feynman → done
- **蒙对标注**（M7.8）：`AttemptRecord.guessed?: boolean`，FeedbackPanel 的「其实是蒙对的」按钮，mastery 计算排除蒙对
- **错题重刷**（NP-12）：独立复习页 `/learn/review/[moduleId]`，使用 volatile `useReviewStore`，打乱顺序出题，结束显示通过率
- **错题本导出**（NP-11）：`wrong-question-book.ts` 导出 Markdown（分「❌ 错题」「🤔 蒙对的题」两节）
- **行内复习**（NP-10）：`collectReviewSlots` 在概念间注入错题+蒙对题作为 `reviewSlots`

### 0.2 问题

| 问题 | 根因 |
|------|------|
| 题库是**扁平命名空间**，无法按主题/学科/项目归类 | `listStoredModules()` 返回按 `updatedAt` 排序的扁平列表，无分组结构 |
| 学完一个题库后无法**自然衔接**下一个相关题库 | done 阶段硬编码重定向到 `/learn/done`，无主题编排层 |
| 错题重刷**局限于单库**，无法跨库聚合薄弱点 | `review-store.startSession(moduleId)` 只接受单个 moduleId，`collectWrongSlotIds` 只扫描一个模块 |
| 蒙对题和错题**混合重刷**，无法针对性复习 | `collectWrongSlotIds` 用 `hasWrong || hasGuessed` 合并收集，无筛选维度 |
| 主题模式下错题重刷**缺乏上下文** | 无主题概念，review 页无法感知主题作用域 |

### 0.3 解决思路

**三层扩展（最小侵入）：**

1. **数据层**：新增 `Topic` 领域模型 + `TopicSession` 会话状态 + reviewFilter 维度
2. **编排层**：新增 `useTopicSessionStore` 在 `ModuleStage` 之上编排模块顺序，拦截 done 重定向
3. **聚合层**：扩展 `review-store` 支持多 moduleId + filter，支持跨库错题/蒙对题聚合重刷

**不侵入单库流程**——从题库卡片直接学习的行为完全不变；主题刷题是显式入口的增强路径。

### 0.4 设计取舍

- **M8.1 不做多对多主题归属。** 12 个题库上限下一对多足够，多对多增加关系表复杂度无收益。如未来题库数量大幅增长再迁移。
- **M8.1 不做主题嵌套（主题下的主题）。** 只有一级主题。树形结构增加 UX 复杂度。
- **M8.1 不做主题级别的费曼任务。** 每个题库走自己的费曼。主题完成度是各模块 mastery 的聚合展示。
- **M8.1 支持主题整体导出/导入。** 新增 `.alc-topic.json` 格式（单 JSON 文件，内含 topic 元数据 + 有序的 `CompiledModulePackage[]`）。与 `.alc-module.json` 风格一致，不引入 zip 依赖。导入时重新分配所有 ID + 创建本地 Topic 关联。
- **M8.1 展示模式与主题功能兼容。** 主题创建/编辑/导出/导入/刷题在展示模式和实用模式下均可用（纯本地操作，无 LLM）。展示模式额外支持从 `manifest.json` v2 加载预配置主题作为 demo 数据。
- **M8.1 不修改行内 reviewSlots 机制。** 概念间错题注入（`collectReviewSlots`）保持 per-module。跨库错题聚合只发生在独立复习页（review-store），不侵入学习流程中的行内复习。
- **M8.1 不做拖拽排序。** 模块在主题中的排序用上/下箭头按钮。拖拽在移动端体验差且实现复杂，12 个题库上限内箭头足够。

## 2. Milestone Scope

### 2.1 Must（M8.1）

1. **Topic 领域模型**：`Topic` interface + `TopicSession` interface + `ReviewFilter` 类型。
2. **主题持久化层**：`topic-library.ts`，CRUD 操作（create / list / update / delete / addModule / removeModule / reorder）。
3. **主题会话 store**：`useTopicSessionStore`（持久化 `alc:state:topic-session`），编排主题内模块顺序。

12. Typecheck、lint、unit tests、Playwright smoke 全部通过。

1. 主题完成时的聚合 mastery 展示（跨模块加权平均 mastery 进度条）。
2. 主题级错题本导出（`downloadWrongQuestionBook` 支持 topic 作用域，按模块分组）。

### 2.3 Not Now（推迟到 V1.2+）

In `src/types/domain.ts`，在 `Module` interface 之后新增：

```ts
// =================================================================
// 主题归类（M8.1）
// =================================================================

/**
 * 主题 — 一组有序的题库引用，用于主题刷题模式。
 *
 * 一对多关系：一个题库最多归属一个主题（由 UI 层保证）。
 * 删除主题不删除题库本身，题库变为未归属。
 */
export interface Topic {
  id: string
  name: string
  description?: string
  /** 有序的题库 ID 列表，决定主题刷题的顺序 */
  moduleIds: string[]
  createdAt: number
  updatedAt: number
}

/**
 * 主题会话中单个题库的状态。
 */
export type ModuleTopicStatus = 'pending' | 'in_progress' | 'done'

/**
 * 主题刷题会话 — 持久化到 LocalStorage，支持刷新恢复。
 *
```

```ts
/**
 * 主题库持久化 — LocalStorage CRUD
 *
 * 全量存储在 alc:topic-index 中（Topic[] 完整序列化）。
 * 预计主题数量 < 10，单次读写完整列表。
 */

import { nanoid } from 'nanoid'

import type { Topic } from '@/types/domain'
import { storage } from './local-storage'
import { StorageKeys } from './keys'

/** 读取所有主题（按 createdAt 升序） */
export function listTopics(): Topic[] {
  const topics = storage.get<Topic[]>(StorageKeys.topicIndex)
  if (!topics || !Array.isArray(topics)) return []
  return [...topics].sort((a, b) => a.createdAt - b.createdAt)
}

/** 读取单个主题 */
export function getTopic(topicId: string): Topic | null {
  return listTopics().find((t) => t.id === topicId) ?? null
}

/** 查询题库所属的主题（一对多：最多一个） */
export function getTopicByModuleId(moduleId: string): Topic | null {
  return listTopics().find((t) => t.moduleIds.includes(moduleId)) ?? null
}

/** 创建主题 */
export function createTopic(name: string, description?: string, moduleIds: string[] = []): Topic {
  const topics = listTopics()
  const topic: Topic = {
    id: `topic-${nanoid()}`,
    name: name.trim(),
    description: description?.trim() || undefined,
    moduleIds,
    createdAt: Date.now(),
    updatedAt: Date.now(),
  }
  // 一对多约束：从其他主题中移除已归属的模块
  const updatedTopics = enforceExclusiveMembership(topics, moduleIds, topic.id)
  updatedTopics.push(topic)
  storage.set(StorageKeys.topicIndex, updatedTopics)
  return topic
}

/** 更新主题元数据（名称、描述） */
export function updateTopic(topicId: string, patch: { name?: string; description?: string }): void {
  const topics = listTopics()
  const idx = topics.findIndex((t) => t.id === topicId)
  if (idx === -1) return
  topics[idx] = {
    ...topics[idx]!,
    ...(patch.name !== undefined ? { name: patch.name.trim() } : {}),
    ...(patch.description !== undefined ? { description: patch.description.trim() || undefined } : {}),
    updatedAt: Date.now(),
  }
  storage.set(StorageKeys.topicIndex, topics)
}

/** 删除主题（不删除题库本身） */
export function deleteTopic(topicId: string): void {
  const topics = listTopics().filter((t) => t.id !== topicId)
  storage.set(StorageKeys.topicIndex, topics)
}

/** 将题库加入主题（一对多：自动从其他主题移除） */
export function addModuleToTopic(topicId: string, moduleId: string): void {
  const topics = listTopics()
  // 先从所有其他主题移除
  const cleaned = enforceExclusiveMembership(topics, [moduleId], topicId)
  // 加入目标主题
  const target = cleaned.find((t) => t.id === topicId)
  if (target && !target.moduleIds.includes(moduleId)) {
    target.moduleIds.push(moduleId)
    target.updatedAt = Date.now()
  }
  storage.set(StorageKeys.topicIndex, cleaned)
}

/** 将题库从主题移除 */
export function removeModuleFromTopic(topicId: string, moduleId: string): void {
  const topics = listTopics()
  const target = topics.find((t) => t.id === topicId)
  if (!target) return
  target.moduleIds = target.moduleIds.filter((id) => id !== moduleId)
  target.updatedAt = Date.now()
  storage.set(StorageKeys.topicIndex, topics)
}

/** 调整模块在主题中的顺序（上/下移动一位） */
export function moveModuleInTopic(topicId: string, moduleId: string, direction: 'up' | 'down'): void {
  const topics = listTopics()
  const target = topics.find((t) => t.id === topicId)
  if (!target) return
  const idx = target.moduleIds.indexOf(moduleId)
  if (idx === -1) return
  const swapWith = direction === 'up' ? idx - 1 : idx + 1
  if (swapWith < 0 || swapWith >= target.moduleIds.length) return
  ;[target.moduleIds[idx], target.moduleIds[swapWith]] = [target.moduleIds[swapWith]!, target.moduleIds[idx]!]
  target.updatedAt = Date.now()
  storage.set(StorageKeys.topicIndex, topics)
}

/** 批量设置主题模块顺序 */
export function reorderModulesInTopic(topicId: string, newModuleIds: string[]): void {
  const topics = listTopics()
  const target = topics.find((t) => t.id === topicId)
  if (!target) return
  target.moduleIds = newModuleIds
  target.updatedAt = Date.now()
  storage.set(StorageKeys.topicIndex, topics)
}

/**
 * 一对多约束执行器：从 topics 中移除 moduleIds 在 otherTopicId 之外的归属。
 * 返回新数组（不修改入参）。
 */
function enforceExclusiveMembership(topics: Topic[], moduleIds: string[], keepTopicId: string): Topic[] {
  return topics.map((t) => {
    if (t.id === keepTopicId) return t
    const hasOverlap = t.moduleIds.some((id) => moduleIds.includes(id))
    if (!hasOverlap) return t
    return {
      ...t,
      moduleIds: t.moduleIds.filter((id) => !moduleIds.includes(id)),
      updatedAt: Date.now(),
    }
  })
}

/**
 * 级联清理：题库被删除时，从所有主题中移除其引用。
 * 由 module-library.ts 的删除逻辑调用。
 */
export function cascadeDeleteModule(moduleId: string): void {
  const topics = listTopics()
  let changed = false
  for (const t of topics) {
    if (t.moduleIds.includes(moduleId)) {
      t.moduleIds = t.moduleIds.filter((id) => id !== moduleId)
      t.updatedAt = Date.now()
      changed = true
    }
  }
  if (changed) storage.set(StorageKeys.topicIndex, topics)
}
```

```ts
/**
 * Topic Session Store — 主题刷题会话（Zustand, 持久化）
 *
 * 编排在 ModuleStage 状态机之上：
 * - startSession(topicId)：快照主题模块顺序，初始化状态映射
 * - markModuleDone(moduleId)：标记当前模块完成
 * - advanceToNextModule()：前进索引，返回下一个 moduleId 或 null（主题完成）
 * - exitSession()：清除会话
 *
 * 持久化到 alc:state:topic-session，支持刷新恢复。
 * 但 startSession 时快照 moduleIds — 用户中途在库页调整顺序不影响进行中的会话。
 */

import { create } from 'zustand'
import { persist, createJSONStorage } from 'zustand/middleware'

import type { ModuleTopicStatus, TopicSession } from '@/types/domain'
import { getTopic } from '@/lib/persistence/topic-library'

interface TopicSessionStoreState {
  session: TopicSession | null

  /** 启动主题会话。返回 true 表示成功，false 表示主题不存在或无模块。 */
  startSession: (topicId: string) => boolean

  /** 标记当前模块为已完成。 */
  markCurrentModuleDone: () => void

  /**
   * 前进到下一个模块。
   * 返回下一个 moduleId，或 null 表示主题已全部完成。
   * 调用方拿到 moduleId 后自行 startModule(moduleId) + router.push。
   */
  advanceToNextModule: () => string | null

  /** 获取当前模块 ID。 */
  getCurrentModuleId: () => string | null

  /** 退出会话（不改变已完成的模块状态）。 */
  exitSession: () => void

  /** 查询是否有活跃会话。 */
  isActive: () => boolean
}

export const useTopicSessionStore = create<TopicSessionStoreState>()(
  persist(
    (set, get) => ({
      session: null,

      startSession: (topicId) => {
        const topic = getTopic(topicId)
        if (!topic || topic.moduleIds.length === 0) return false

        const moduleStatus: Record<string, ModuleTopicStatus> = {}
        for (const mid of topic.moduleIds) {
          moduleStatus[mid] = 'pending'
        }
        const firstModuleId = topic.moduleIds[0]!
        moduleStatus[firstModuleId] = 'in_progress'

        set({
          session: {
            topicId,
            moduleIds: [...topic.moduleIds], // 快照
            currentIndex: 0,
            moduleStatus,
            startedAt: Date.now(),
          },
        })
        return true
      },

      markCurrentModuleDone: () => {
        set((state) => {
          if (!state.session) return state
          const currentId = state.session.moduleIds[state.session.currentIndex]
          if (!currentId) return state
          return {
            session: {
              ...state.session,
              moduleStatus: {
                ...state.session.moduleStatus,
                [currentId]: 'done',
              },
            },
          }
        })
      },

      advanceToNextModule: () => {
        const state = get()
        if (!state.session) return null
        const nextIndex = state.session.currentIndex + 1
        if (nextIndex >= state.session.moduleIds.length) return null

        const nextModuleId = state.session.moduleIds[nextIndex]!
        set({
          session: {
            ...state.session,
            currentIndex: nextIndex,
            moduleStatus: {
              ...state.session.moduleStatus,
              [nextModuleId]: 'in_progress',
            },
          },
        })
        return nextModuleId
      },

      getCurrentModuleId: () => {
        const session = get().session
        if (!session) return null
        return session.moduleIds[session.currentIndex] ?? null
      },

      exitSession: () => set({ session: null }),

      isActive: () => get().session !== null,
    }),
    {
      name: 'alc:state:topic-session',
      storage: createJSONStorage(() => localStorage),
    },
  ),
)
```

```ts
/**
 * 跨库错题聚合 — 扫描多个模块，收集符合 filter 的 slotId + Quiz。
 */

import type { AttemptRecord, Module, Quiz, ReviewFilter } from '@/types/domain'
import { findQuizInModule } from './adaptive-sequencer'

const PASS_THRESHOLD = 80

/**
 * 判断单个 slot 是否匹配 filter 条件。
 */
export function matchesFilter(
  attempts: AttemptRecord[] | undefined,
  filter: ReviewFilter,
): boolean {
  if (!attempts || attempts.length === 0) return false
  const hasWrong = attempts.some((a) => a.score < PASS_THRESHOLD)
  const hasGuessed = attempts.some((a) => a.guessed === true)
  switch (filter) {
    case 'wrong':
      return hasWrong
    case 'guessed':
      return hasGuessed
    case 'all':
      return hasWrong || hasGuessed
  }
}

export interface CollectedReviewItem {
  moduleId: string
  slotId: string
  quiz: Quiz
}

/**
 * 扫描多个模块，收集符合 filter 的 review items。
 */
export function collectReviewItemsForModules(
  modules: Module[],
  attemptsBySlot: Record<string, AttemptRecord[]>,
  filter: ReviewFilter,
): CollectedReviewItem[] {
  const items: CollectedReviewItem[] = []

  for (const module of modules) {
    const allQuizzes: Quiz[] = [
      ...module.concepts.flatMap((c) => c.quizSeries.quizzes),
      ...(module.challengeQuizzes ?? []),
    ]
    for (const quiz of allQuizzes) {
      if (matchesFilter(attemptsBySlot[quiz.id], filter)) {
        items.push({ moduleId: module.id, slotId: quiz.id, quiz })
      }
    }
  }

  return items
}
```

> **注意：** `findQuizInModule` 在现有 `adaptive-sequencer.ts` 中已存在。这里直接内联遍历 quiz 列表更直观（跨库场景需要知道 quiz 属于哪个模块）。

- [ ] **Step 1: CompiledTopicPackage 类型与序列化**

Create `src/lib/persistence/topic-package.ts`:

```ts
/**
 * Compiled Topic Package — 主题整体导出/导入格式
 *
 * 单 JSON 文件，内含 topic 元数据 + 有序的 CompiledModulePackage[]。
 * 导入时重新分配所有 ID（module + source + quiz），
 * 并创建本地 Topic 关联新的 moduleId 列表。
 *
 * 与 CompiledModulePackage 安全规范一致：
 * - 不包含 apiKey
 * - 每个 module 块通过 parseModulePackage 6 步校验
 */

import { nanoid } from 'nanoid'

import type { Topic } from '@/types/domain'
import type { CompiledModulePackage } from './module-package'
import {
  parseModulePackage,
  importModulePackage,
  createModulePackage,
  serializeModulePackage,
} from './module-package'
import { createTopic } from './topic-library'
import type { StorageRepository } from './repository'

export interface CompiledTopicPackage {
  version: 1
  exportedBy: 'ai-learning-compiler'
  exportedAt: number
  topic: {
    name: string
    description?: string
  }
  /** 有序模块数组，顺序 = 主题刷题顺序 */
  modules: CompiledModulePackage[]
}

export type ParseTopicPackageResult =
  | { ok: true; pkg: CompiledTopicPackage }
  | { ok: false; error: string }

// =================================================================
// 导出
// =================================================================

export function createTopicPackage(args: {
  topic: Topic
  modulePackages: CompiledModulePackage[]
}): CompiledTopicPackage {
  return {
    version: 1,
    exportedBy: 'ai-learning-compiler',
    exportedAt: Date.now(),
    topic: {
      name: args.topic.name,
      description: args.topic.description,
    },
    modules: args.modulePackages,
  }
}

export function serializeTopicPackage(pkg: CompiledTopicPackage): string {
  return JSON.stringify(pkg, null, 2)
}

// =================================================================
// 解析（导入前校验）
// =================================================================

export function parseTopicPackage(raw: unknown): ParseTopicPackageResult {
  if (!raw || typeof raw !== 'object' || Array.isArray(raw)) {
    return { ok: false, error: '主题包格式无效：期望对象' }
  }
  const obj = raw as Record<string, unknown>

  if (obj.version !== 1) {
    return { ok: false, error: `不支持的主题包版本：${String(obj.version)}` }
  }
  if (obj.exportedBy !== 'ai-learning-compiler') {
    return { ok: false, error: '主题包来源不明' }
  }
  if (!obj.topic || typeof obj.topic !== 'object') {
    return { ok: false, error: '主题包缺少 topic 元数据' }
  }
  const topicMeta = obj.topic as Record<string, unknown>
  if (typeof topicMeta.name !== 'string' || topicMeta.name.trim() === '') {
    return { ok: false, error: '主题名称无效' }
  }

  if (!Array.isArray(obj.modules) || obj.modules.length === 0) {
    return { ok: false, error: '主题包模块列表为空' }
  }

  // 安全检查：整个 JSON 字符串不含 apiKey
  const jsonStr = JSON.stringify(raw)
  if (jsonStr.includes('apiKey')) {
    return { ok: false, error: '主题包包含敏感信息（apiKey），已拒绝' }
  }

  // 逐个校验模块包
  for (let i = 0; i < obj.modules.length; i++) {
    const moduleResult = parseModulePackage(obj.modules[i])
    if (!moduleResult.ok) {
      return { ok: false, error: `模块 #${i + 1} 校验失败：${moduleResult.error}` }
    }
  }

  // 重新解析以获得类型安全的数据（parseModulePackage 已验证每个模块）
  return { ok: true, pkg: raw as CompiledTopicPackage }
}

// =================================================================
// 导入
// =================================================================

/**
 * 导入主题包：逐个导入模块 + 创建本地 Topic。
 *
 * 每个模块通过 importModulePackage 分配新 ID（nanoid），
 * 然后用 createTopic 关联新的 moduleId 列表。
 *
 * @returns 新创建的 Topic
 */
export function importTopicPackage(repo: StorageRepository, pkg: CompiledTopicPackage): Topic {
  const newModuleIds: string[] = []

  for (const modulePkg of pkg.modules) {
    const module = importModulePackage(repo, modulePkg)
    newModuleIds.push(module.id)
  }

  return createTopic(pkg.topic.name, pkg.topic.description, newModuleIds)
}

// =================================================================
// 浏览器下载
// =================================================================

export function downloadTopicPackage(pkg: CompiledTopicPackage): void {
  const json = serializeTopicPackage(pkg)
  const blob = new Blob([json], { type: 'application/json' })
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')
  a.href = url
  a.download = `${pkg.topic.name}.alc-topic.json`
  document.body.appendChild(a)
  a.click()
  document.body.removeChild(a)
  URL.revokeObjectURL(url)
}
```

```ts
import { createTopic } from '@/lib/persistence/topic-library'
import type { Topic } from '@/types/domain'

/**
 * 加载展示主题到 LocalStorage。
 *
 * 流程：
 * 1. 按顺序 fetch 每个模块包 JSON
 * 2. parseModulePackage 校验
 * 3. importModulePackage 分配新 ID + 写入 storage
 * 4. createTopic 关联所有新 moduleId
 *
 * 每次调用分配新 ID，用户可反复加载同一主题（每次产生新副本）。
 *
 * @returns 新创建的本地 Topic
 */
export async function loadShowcaseTopicIntoStorage(
  entry: ShowcaseTopicEntry,
): Promise<Topic> {
  const newModuleIds: string[] = []

  for (const modEntry of entry.modulePackages) {
    const rawPackage = await fetchShowcasePackage(modEntry.package)
    const parsed = parseModulePackage(rawPackage)
    if (!parsed.ok) {
      throw new Error(`展示主题模块加载失败：${modEntry.package} — ${parsed.error}`)
    }
    const module = importModulePackage(storage, parsed.pkg)
    newModuleIds.push(module.id)
  }

  return createTopic(entry.name, entry.description, newModuleIds)
}
```

## 6. 风险登记与缓解

| Risk | Impact | Mitigation |
|---|---|---|
| topic-session-store 持久化后，用户在模块学习中刷新页面 | session 保留，但 progress-store 也持久化（per-module），恢复后继续当前模块。过渡页可正常 resume | 现有 progress-store 已有 per-module sync subscriber，天然支持恢复 |
| 用户在主题会话进行中，在库页修改了主题的模块顺序 | session 内的 `moduleIds` 是快照，不受库页修改影响。已完成的模块状态不受影响 | 设计预期：会话内顺序固定，修改顺序下次会话生效 |
| 用户在主题会话进行中，删除了主题中尚未学习的模块 | 过渡页加载下一个模块时 `loadStoredModule` 返回 null → 退出会话 + 提示 | `handleContinue` 中 null 检查 + `exitSession` fallback |
| 跨库 review 的 Quiz ID 冲突 | Quiz ID 格式为 `{moduleId}:{slotIndex}`，nanoid 分配的 moduleId 保证全局唯一，不会跨库冲突 | 现有 ID 分配机制（`assignLocalModuleIdentity`）已保证 |
| 一对多约束在并发编辑时失效 | LocalStorage 无事务，极端情况下两个 tab 同时操作可能不一致 | UI 层单次操作原子读写全量 topic-index；可接受概率极低的数据不一致 |
| 过渡页的模块状态依赖 progress-store 的 per-module 数据 | `moduleStatus` 存在 session 中，但实际 mastery/attempts 在全局 store。过渡页显示 mastery 时需从 `alc:mastery:{moduleId}` 读取 | 过渡页通过 `computeMastery` + `loadStoredModule` 实时计算 |
| review-store 的 ReviewSession 结构变更破坏现有 review 页 | `queue` 从 `Quiz[]` 变为 `ReviewQueueItem[]`，现有 review 页的 `session.queue[currentIndex]` 访问方式变化 | 同步修改 review 页；抽取 ReviewQueueItem.quiz 的访问；typecheck 兜底 |
| 主题级 review 写入 attempts 后影响各模块 mastery | review attempts 有 `attemptVersion > 0`，mastery 只看 `attemptVersion === 0`，不受影响 | 现有 mastery 设计天然隔离 review attempts |
| 删除主题后 topic-session-store 残留 | session 指向已删除的 topicId | 过渡页检测 `getTopic(topicId)` 返回 null 时自动 `exitSession` + 跳库页 |
