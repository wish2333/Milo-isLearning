# V2.1.4 Developing — 费曼练习 UX 与答案反馈统一

> 状态：Done | 日期：2026-07-22
> 前序基线：V2.1.3（错题编辑 Amnesty、production storage 兼容、849 tests）
> 关闭边界：费曼步骤纠题与持久化、费曼作答历史、选择题正确答案三态、Today 复习编辑入口、全量验证

## 0. 背景

V2.1.3 收尾后的构建实测继续发现一组相互关联的问题：

1. 费曼步骤答错后没有“编辑此题”入口。
2. 编辑入口与解析文字视觉层级相同，难以发现。
3. 费曼步骤仍使用“回看上一题”旧交互，无法查看完整作答历史。
4. 费曼答案修改没有独立的 Module 持久化路径。
5. 费曼及部分旧入口没有稳定显示“正确答案 ✓ / 用户错选 ✗”；概念题需要统一三态表现。
6. Today 复习反馈入口没有复用题目编辑与重评链路。

## 1. 实施内容

### 1.1 费曼步骤编辑与持久化

- `module-store` 新增 `updateFeynmanStep`，只允许 user Module 写入 `alc:module:*`。
- `FeynmanStepView` 将步骤包装成既有 `AnswerCorrector` 所需的 Quiz 形状，复用选择题/填空题编辑器。
- 编辑后立即用当前用户答案重评，并同步 `feynmanAttempt.stepResults`；修正正解后当前反馈立即变为答对。
- Feynman attempt 记录新增可选 `userAnswer`，兼容旧记录。

### 1.2 费曼 UI/UX 对齐

- 新增 `FeynmanHistoryPanel`，以“作答历史”替代“回看上一题”。
- 历史面板展示步骤、用户作答、得分状态与参考答案，不改变当前学习位置。
- 费曼最终任务页同样提供作答历史入口。
- 纠题操作区增加分隔线、强调色边框与背景，和解析内容形成明确视觉区分。

### 1.3 选择题反馈统一

- `ChoiceQuiz` 在提交态始终渲染正确项 ✓。
- 用户错选项渲染 ✗；其它选项降级显示。
- 兼容未透传 `submittedAnswer` 的旧调用方，使用组件内部保留的 selected 值恢复三态反馈。
- Feynman 入口显式透传 submitted answer，确保费曼与概念题一致。

### 1.4 Today 复习自查修复

- Today 复习页补齐题目编辑入口、Module 注入、答案重评与 schedule 同步。
- TodaySession 新增 `updateResult`，纠题后同步更新今日结果分数与通过状态。
