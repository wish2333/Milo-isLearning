# V2.1.5 Developing — 作答进度联动与重刷界面收尾

> 状态：Done | 日期：2026-07-22
> 前序基线：V2.1.4（费曼练习 UX、答案反馈与纠题持久化）
> 需求来源：[`v2.1.4-fix-report2.md`](./v2.1.4-fix-report2.md) 与补充 bug5、bug6
> 关闭边界：主题进度联动、复习槽位恢复、模块顺序与进度展示、重刷导航与背景信息、题库主题折叠、完成页防御、全流程底部固定操作栏、双模式构建与浏览器回归

## 0. 背景

V2.1.4 之后的审查发现，主题学习的“作答 → 落盘 → 恢复 → 展示”链路仍有断点；同时重刷页沿用了旧 UI 壳层，造成主题重刷双层导航、题目前背景丢失，以及解析出现后下一题按钮被推离视口。补充 bug5 将这些问题归并为一次 UI 与状态联动收尾。

## 1. 实施内容

### 1.1 主题进度联动

- `topic-session-store` 抽出统一 `persistTopicProgress`，完成、推进、跳过、重新进入均立即写入 TopicProgress，不再依赖退出主题。
- 同一主题存在有效 `in_progress` 会话时复用内存会话，避免从题库页重复点击导致进度重建；全部完成后仍保留从头开始语义。
- `TopicTransitionView` 用模块级 `ProgressState` 对主题状态做展示兜底：`done` 显示已完成，非初始阶段至少显示进行中。
- 新增恢复、advance/skip/reenter 落盘与状态合并测试。
- 统一生产模式的当前 repository：题库、主题、模块进度、主题会话与首页摘要均从 SQLite-backed `getStorage()` 读取，不再把浏览器 LocalStorage 当作当前真值。
- 对 V2.1.3 之前误写在浏览器 LocalStorage 的模块/主题记录增加兼容回填；主题索引支持与 SQLite 中已有索引合并，避免旧会话指向的主题被误判为不存在。
- 题库列表枚举生产数据时同时纳入旧 LocalStorage 的 Module key，并按需回填 Module 与其进度，避免主题只渲染出已经迁入 SQLite 的少数模块。
- 修复生产模式下导入、主题扩充、批量扩充、错题恢复与完成页仍直接使用旧 LocalStorage 的写入路径；统一改走 SQLite-backed repository。
- production 启动时自动补齐旧 LocalStorage 中缺失的用户数据；主编译完成后等待写队列及自动重试结束，并在 Module/Source/Quality 任一落盘失败时阻止跳转。
- 写队列的 `flushNow()` 现在会等待退避重试完成，避免编译结果只停留在浏览器缓存而未进入 SQLite。

### 1.2 题库与首页导航

- 首页最近学习模块改用与题库页一致的 repository，避免 production 下 `updatedAt` 全为 0 后退化为存储插入顺序。
- 主题详情默认只保留主题摘要；模块列表支持折叠/展开，默认全部收起，减少多主题页面的长距离滚动。
- 去掉主题重刷页额外的 `LearnShell`，统一由 `/learn/layout` 提供单一 `GlobalNav`，修复双层导航栏。

### 1.3 bug6：模块复习队列与进度真值

- 复习槽位在恢复、进入下一概念和推进时统一按当前 Module 过滤，清理题目被编辑/忽略/删除后遗留的无效 ID，并去重。
- 旧快照的 `quizIndex` 会按有效 `originalQuizId` 重新映射；游标落在无效尾槽位时自动推进，不再停留在“题目加载中”。
- Concept 页面不再接受跨题、跨模块残留的 `currentQuiz`；只有当前槽位的替换题或原题可以渲染，作答记录继续以稳定的 `originalQuizId` 归档。
- 主题与题库列表订阅进度/作答历史刷新信号，展示“题目 n/总数”“复习题 n/总数”等当前位置，避免摘要与作答历史脱节。
- 主题卡不再只用已完成模块数判断“未开始”，会将模块级 `ProgressInfo.started` 合并为“进行中”，并保留模块完成数与进行中数量的区分。

### 1.4 题目呈现与固定操作区

- 单库重刷、主题重刷、Today 复习恢复题目前背景 `BackgroundPanel`。
- 新增 `QuizActionBar`，将确认选择/确认答案/确认排序、解析后的下一题/继续/换题、费曼最终提交固定在视口底部。
- 固定操作区使用半透明背景、模糊、阴影和 safe-area inset，并为页面内容预留底部空间，兼容移动端手势区。

### 1.5 完成页防御

- 完成页对异常或旧格式快照增加 `computeMastery` 防御，避免 `/learn/done` 渲染阶段直接抛出客户端异常。

### 1.6 协作与 review

- 使用 `gpt-5.6-luna` high reasoning 子代理独立完成主题状态链路调研与限定文件实现。
- 主代理复核会话复用边界、全部完成后的重开语义、存储兼容性及 UI 交叉影响后整合。

## 3. 已知边界

1. 自动回填只会把旧 LocalStorage 中 SQLite 缺失的用户数据补入数据库，已有 SQLite 数据优先；旧 LocalStorage 保留，便于恢复与审计，暂不自动清理。
2. 没有用户提供 `/learn/done` 的实际红色堆栈，因此本版提供渲染防御层，若线上仍出现异常需结合堆栈继续定位。
3. Next.js 仍提示工作区存在多个 lockfile；不影响构建结果。
