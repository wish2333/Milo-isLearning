### 1.4 P0.4 — pre-existing E2E 失败修复（✅ Done）

**实际根因（3 处，全部修复）**：

1. **Playwright `reuseExistingServer: true` 复用本地 production server**：用户本地若已启动 `bun run start` 的 production 构建服务器（port 3000），Playwright 会直接连上去。该服务器 `isShowcaseMode=false`，导致 `AppShell` 渲染 `StorageLoading`（"正在加载本地学习数据..."）永不水合，所有子页面被阻塞。**修复**：`playwright.config.ts` 端口 3000 → 3002，避免冲突。
2. **两个 `next dev` server 共享 `.next/` 缓存目录**：chromium (@3002) + chromium-showcase (@3001) 两个项目同时跑 `bun run dev`，共享 `.next/` 导致编译产物互相破坏，showcase 测试偶发失败。**修复**：`next.config.ts` 条件化 `distDir`（读 `E2E_CACHE_DIR` env），两个 server 分别用 `.next-3001/` `.next-3002/`。
3. **`FeynmanFinalView.handleSubmit` 在 `config` 为 null 时静默 return**：`Topic learning flow` 测试的 `beforeEach` 未注入 LLM config 到 localStorage，导致点击"提交评估"无反应（既不报错也不调 `/api/feynman-eval`），最终 timeout。**修复**：`e2e/topic.spec.ts` 的 Topic learning flow `beforeEach` 加 `addInitScript` 注入 mock `alc:settings`。

**修复涉及文件**：
- `playwright.config.ts`（端口 3002 + `testIgnore: /showcase/` + `E2E_CACHE_DIR`）
- `next.config.ts`（条件化 `distDir`）
- `e2e/topic.spec.ts`（Topic learning flow `beforeEach` 加 mock LLM config）
- `e2e/showcase/showcase-home.spec.ts`（删除 1 行冗余注释，无实质改动）
- `.gitignore`（加 `.next-*/`）
- `eslint.config.mjs`（加 `.next-*/**` 到 ignores，避免 lint 跑 E2E 临时构建目录产生 5386 个 false-positive）

**Acceptance 满足**：全量 E2E 48 passed / 8 skipped（production storage-layer 自跳过）/ 0 failed。V2.0.1 re-entry A/B 用例仍通过，无新回归。

| 4 | per-module storage 二分（架构债） | **推迟 V2.0.2**（独立 milestone） | V2.1.0 P1 plan 已要求"兼容当前双写路径"（约束已计划）。统一 backend 失败模式是"用户进度丢失"（最高风险），需要数据迁移 + 回退方案 + 全量回归。与 P1 批量扩充交织会让复杂度指数级上升。Effort: Large (3d+) |
