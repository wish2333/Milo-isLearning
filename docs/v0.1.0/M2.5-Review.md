## 2. 真实 LLM Smoke Run 结果

执行环境：`.env.local` 已配置 `GLM_API_KEY`。脚本通过 `bun --env-file=.env.local run scripts/prompt-eval.ts` 运行（bun 默认不加载 .env.local，需显式指定）。

### 2.1 import agent（轻量级 Agent）

| 组合 | runs | 成功 | Retry | P50 | P95 |
|------|------|------|-------|-----|-----|
| glm-5-turbo / edge/very-short | 1 | 1/1 (100%) | 0/1 | 2979ms | 2979ms |
| glm-5-turbo / rag-medium | 1 | 1/1 (100%) | 0/1 | 4349ms | 4349ms |

结论：turbo 适合 import agent。M3 默认 `DEFAULT_LIGHTWEIGHT_MODEL = glm-5-turbo`。

### 2.2 concept agent（编译主体代表）

| 组合 | runs | 成功 | Retry | P50 | P95 |
|------|------|------|-------|-----|-----|
| glm-5-turbo / rag-chunks | 3 | 3/3 (100%) | 0/3 | 16637ms | 17195ms |

结论：concept agent 在 turbo 上 Schema 通过率 100%，但单次延迟 17s 偏高（M3 编译主体 6 个 Agent 串行将超 P95 ≤ 60s 红线）。M3 应改用 `glm-5.2`。

### 2.3 chunk agent（编译主体代表）

| 组合 | runs | 成功 | Retry | P50 | P95 |
|------|------|------|-------|-----|-----|
| glm-5-turbo / rag-medium | 2 | 0/2 (0%) | 0/2 | 61055ms | 61092ms |

失败原因：`ProviderError: request timeout after 30000ms`。

结论：turbo 处理 2k 字符 md 输入 + 结构化 JSON 输出会超时（30s）。M3 编译主体不能用 turbo。

| Retry 触发率 | ≤ 30% | 全部 0% | 达标 |

## 3. 关键工程发现

### 3.1 bun 不自动加载 .env.local

M1 ping 脚本能跑是因为人工设置过 shell 环境变量。`bun run scripts/...` 默认只读 process.env，不加载 .env.local。

**M3 行动项**：所有需要 API key 的脚本与 dev server，文档说明使用 `bun --env-file=.env.local`，或考虑添加 `bunfig.toml` 启用 env loading。

### 3.2 turbo 模型不适合结构化中等负载

`glm-5-turbo` 在 2k 字符 md → JSON 输出的 chunk agent 上 100% 超时。这与 Prompt-Eng §12.6 的预期一致（turbo 思考能力弱，结构化输出场景应用旗舰）。

**M3 行动项**：`pipeline.ts` 区分两组模型：
```ts
const DEFAULT_COMPILE_MODEL = 'glm-5.2'       // Chunk/Concept/Module/Mission/Quiz/Feynman
const DEFAULT_LIGHTWEIGHT_MODEL = 'glm-5-turbo' // Import/Feedback/Feynman-Eval
```

### 3.3 extraBody 浅合并策略

`openai-compat.ts` 把 `extraBody` 字段 shallow-merge 进请求 body，且**不覆盖已存在的键**（model/messages/...）。这意味着调用方若传 `extraBody: { model: 'x' }` 不会生效——安全默认。

测试已覆盖：`extraBody does NOT overwrite reserved fields`。

### 3.4 映射层比预期更薄

M1-Review §6.1 设想 9 个 Agent 都需要 snake_case → camelCase 转换。实际盘点后发现：
- 只有 `feedback` 用了 snake_case（`next_action` / `feedback_text`）
- 其余 8 个 Agent 字段命名已是 camelCase
- 枚举字符串 `'fill_blank'` 是值不是字段名，无需转换

但 `mappers.ts` 仍提供了 9 个函数 + 4 个 assemble 帮手，便于 M3 `assemble()` 统一调用，且未来若 Schema 改名只改一处。
