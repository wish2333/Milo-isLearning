# M7.5 Implementation Plan — Compiled Module Workspace

> **For agentic workers:** REQUIRED SUB-SKILL: Use `subagent-driven-development` or `executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 M7 已经能生成的 Learning Module 变成可保存、可恢复、可导出、可复用、可回看的产品资产，并完成 UI 视觉底座对齐。

**Architecture:** M7.5 采用 local-first 方案，不引入账号、数据库或服务端持久化。Module / source / progress 继续使用现有 LocalStorage repository；导出导入使用版本化 JSON package；编译刷新恢复先保证“不丢上下文 + 可重启/恢复界面”，真正跳过已完成 Agent 阶段的续编作为 M7.6。

**Tech Stack:** Next.js App Router, React 19, TypeScript strict, Zustand persist, LocalStorage repository, Zod, Vitest, Playwright, Tailwind CSS.

## Global Constraints

- 不引入服务端持久化，不增加账号系统，不做云同步。
- 不把 API Key 写入导出文件、质量报告、URL、日志或 package metadata。
- 不收紧 M7 已放宽的 LLM schema；M7.5 质量报告只观察，不阻断编译。
- 不重写全部 UI；本阶段只落地 design token、核心页面基础样式和可读性修正。
- 导入 JSON 必须跳过 `/api/compile`，这是节省 LLM 成本的核心验收。
- 刷新恢复的产品文案必须诚实：M7.5 保证不丢源文本和恢复入口；按 stage 续编另列 M7.6。
- 所有删除操作必须二次确认。

## 0. 审查修订摘要

原 V1.0 方向正确，但有 6 个需要修正的风险点：

1. **范围过大**：Library、导入导出、IndexedDB checkpoint、stage resume、attempts 归档、UI 改造、质量报告一次全做，已经超出“内测前工程加固”的合理范围。
2. **编译续跑承诺过强**：当前 pipeline 的 stage output 没有作为可恢复输入建模，直接做 `resumeFrom + stageOutputs` 会牵动 `compileMarkdown()` 的主结构。M7.5 先做恢复入口和源文本不丢；M7.6 再做真正跳过已完成 stage。
3. **attempts 存储判断不准**：代码实际用 `alc:state:attempts` 存 active attempts，不是 `StorageKeys.attempts(quizId)`。计划里按 `alc:attempts:*` 清理会失效。
4. **source bug 测试夹具会掩盖 bug**：现有 `quota.test.ts` 的 `seedModule()` 用 `StorageKeys.source(moduleId)` 保存 source，正好匹配了错误删除逻辑。测试必须改成用 `module.sourceId` 保存 source。
5. **UI grep 阈值不现实**：`neutral- <= 10` 会把实现变成机械改 class。改成核心页面 token 覆盖 + 禁止新写大面积 `neutral-*` 主视觉。
6. **新增测试数量不应硬编码**：从“新增 ≥ 30 case”改为每个任务有针对性测试，出口跑 typecheck / lint / test / e2e。

```ts
import type { Module, ProgressState } from '@/types/domain'
import { StorageKeys } from './keys'
import type { StorageRepository } from './repository'

export interface StoredModuleSummary {
  id: string
  sourceId: string
  title: string
  conceptCount: number
  quizCount: number
  updatedAt: number
  completed: boolean
}

export function listStoredModules(repo: StorageRepository): StoredModuleSummary[] {
  return repo
    .keys()
    .filter((key) => key.startsWith('alc:module:'))
    .map((key) => repo.get<Module>(key))
    .filter((module): module is Module => module !== null)
    .map((module) => {
      const progress = repo.get<ProgressState>(StorageKeys.progress(module.id))
      const quizCount =
        module.concepts.reduce((sum, concept) => sum + concept.quizSeries.quizzes.length, 0) +
        (module.challengeQuizzes?.length ?? 0) +
        module.feynmanTask.steps.length
      return {
        id: module.id,
        sourceId: module.sourceId,
        title: module.title,
        conceptCount: module.concepts.length,
        quizCount,
        updatedAt: progress?.updatedAt ?? 0,
        completed: progress?.stage.kind === 'done',
      }
    })
    .sort((a, b) => b.updatedAt - a.updatedAt)
}

export function loadStoredModule(repo: StorageRepository, moduleId: string): Module | null {
  return repo.get<Module>(StorageKeys.module(moduleId))
}

export function resetStoredModuleProgress(repo: StorageRepository, moduleId: string): void {
  repo.remove(StorageKeys.progress(moduleId))
  repo.remove(StorageKeys.feynman(moduleId))
  repo.remove(StorageKeys.attemptsModule(moduleId))
}
```

```ts
import { nanoid } from 'nanoid'
import type { KnowledgeSource, Module } from '@/types/domain'
import { StorageKeys } from './keys'
import type { StorageRepository } from './repository'

export interface CompiledModulePackage {
  version: 1
  exportedBy: 'ai-learning-compiler'
  exportedAt: number
  source: KnowledgeSource
  module: Module
  qualityReport?: unknown
  generatedBy?: {
    provider?: string
    model?: string
    generatedAt?: number
  }
}

export type ParsePackageResult =
  | { ok: true; pkg: CompiledModulePackage }
  | { ok: false; error: string }

export function createModulePackage(args: {
  source: KnowledgeSource
  module: Module
  qualityReport?: unknown
  generatedBy?: CompiledModulePackage['generatedBy']
}): CompiledModulePackage {
  return {
    version: 1,
    exportedBy: 'ai-learning-compiler',
    exportedAt: Date.now(),
    source: args.source,
    module: args.module,
    qualityReport: args.qualityReport,
    generatedBy: args.generatedBy,
  }
}

export function serializeModulePackage(pkg: CompiledModulePackage): string {
  return JSON.stringify(pkg, null, 2)
}

export function parseModulePackage(json: string): ParsePackageResult {
  let parsed: unknown
  try {
    parsed = JSON.parse(json)
  } catch {
    return { ok: false, error: '文件不是合法 JSON' }
  }

  if (!parsed || typeof parsed !== 'object') {
    return { ok: false, error: '文件内容不是对象' }
  }
  const pkg = parsed as Partial<CompiledModulePackage>
  if (pkg.version !== 1) return { ok: false, error: '不支持的导出版本' }
  if (pkg.exportedBy !== 'ai-learning-compiler') {
    return { ok: false, error: '不是 AI Learning Compiler 导出文件' }
  }
  if (!pkg.source || !pkg.module) return { ok: false, error: '导出文件缺少 source 或 module' }
  if (JSON.stringify(pkg).includes('"apiKey"')) {
    return { ok: false, error: '导出文件不能包含 API Key' }
  }
  return { ok: true, pkg: pkg as CompiledModulePackage }
}

export function importModulePackage(repo: StorageRepository, pkg: CompiledModulePackage): Module {
  const nextSourceId = `source-${nanoid()}`
  const nextModuleId = `module-${nanoid()}`
  const module: Module = {
    ...pkg.module,
    id: nextModuleId,
    sourceId: nextSourceId,
    importedAt: Date.now(),
    concepts: pkg.module.concepts.map((concept) => ({
      ...concept,
      moduleId: nextModuleId,
    })),
    feynmanTask: {
      ...pkg.module.feynmanTask,
      moduleId: nextModuleId,
    },
  }
  repo.set(StorageKeys.source(nextSourceId), {
    ...pkg.source,
    id: nextSourceId,
    createdAt: Date.now(),
  })
  repo.set(StorageKeys.module(nextModuleId), module)
  if (pkg.qualityReport) repo.set(StorageKeys.qualityReport(nextModuleId), pkg.qualityReport)
  return module
}
```

## 4. Acceptance Criteria

| Area | Acceptance |
|---|---|
| Storage cleanup | `removeModule()` deletes module, source by `sourceId`, progress, feynman, quality report, module attempts archive |
| Library | `/learn/library` lists stored modules after refresh |
| Open/continue | Library can open a stored module without recompiling |
| Export | Exported JSON includes source/module/version and never includes API Key |
| Import | Import JSON writes a new module/source id and skips `/api/compile` |
| Cost reuse | Imported package can be learned immediately with no LLM compile call |
| Compile recovery V1 | Refreshing compiling page shows latest saved source context and recovery actions |
| Previous review | User can view previous question answer/explanation without changing progress |
| UI token | Core pages use `--bg-base`, `--fg-primary`, `--accent-primary`, success/warning tokens |
| Quality summary | Library can display quiz count, expression distribution, challenge coverage |
| Verification | `bun run typecheck`, `bun run lint`, `bun run test`, and relevant Playwright specs pass |

## 5. Deferred M7.6: True Stage Resume

M7.5 intentionally does not implement true stage resume. M7.6 should start only after M7.5 asset layer lands.

Required design work for M7.6:

1. Refactor `compileMarkdown()` so each stage can accept prior outputs as typed inputs.
2. Add `CompileCheckpoint` Zod schemas for import/chunk/concept/module/mission/quiz/challenge/feynman outputs.
3. Persist stage outputs in IndexedDB, not LocalStorage.
4. Add `resumeFrom` request support to `/api/compile`.

This is a separate milestone because it changes the compiler architecture, not just the frontend recovery experience.
