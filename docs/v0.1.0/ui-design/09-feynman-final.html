示例范文

RAG（Retrieval-Augmented Generation，检索增强生成）通过检索外部知识，把它和用户问题一起喂给模型。 这样模型在生成答案时既能引用最新或私有知识，又能避免凭空捏造（幻觉）。 与微调不同，RAG 不更新模型参数，只在推理时提供上下文，因此适合频繁更新的知识。 RAG 的局限在于检索质量：如果检索不到相关文档，模型生成的答案仍然会出现幻觉。
