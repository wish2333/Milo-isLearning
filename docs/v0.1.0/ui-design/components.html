§4 · Option System

Choice 选项 4 状态变体（默认 / Hover / Selected / Correct / Wrong）。 答对用 ::after + transform: scaleX() 划过 1px 细线（点击下方任意选项试试）。 答错用陶土橙（无 ✗ 符号），同步亮起正确选项的鼠尾草绿。

option-choice · 交互演示（点击切换状态）

点击选项触发"答对"细线动效；点击 [演示答错] 触发"答错"反馈。

§7 · Progress Indicators

阶梯是唯一贯穿全局的视觉隐喻。Concept 内 Quiz 进度用 1px 细线条辅助。 Feynman 步骤用 6 圆点指示器（active 圆点带柔和光晕，是唯一允许外发光的元素）。

  5. C

staircase · 全部完成态（含一个 skipped，与 completed 几乎无差异）
  1. A
  2. A

第 3 格（B-1）是"失败但跳过"状态，与已完成格在视觉上几乎无差异—— 避免给用户制造"我留下了瑕疵"的负面感（M3 错误即学习）。

§8 · Feedback Components

解释面板用 grid-template-rows: 0fr → 1fr 丝滑展开（V1.1 修订， 性能优于 max-height）。反馈文案 ≤ 50 字，鼓励性语气，绝不出现感叹号、emoji、"恭喜"。

panel-explanation · 解释面板（点击按钮切换展开/收起）
很好。
Few-shot 通过提供示例引导模型输出格式，让模型模仿示例的结构。 其他选项中，B 是 zero-shot（无示例），C 是 RLHF（基于人类反馈的强化学习）， D 是 fine-tuning（参数微调）。注意 RAG 与 fine-tuning 的本质差异： RAG 注入上下文，fine-tuning 更新参数。
