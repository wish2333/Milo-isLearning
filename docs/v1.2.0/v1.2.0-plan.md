### 2.3 核心设计决策

#### D-1：F24 跳过用新 `'skipped'` 状态（非复用 pending/done）

**选项**：A) 跳过后保持 `'pending'`，仅推进 currentIndex / B) 新增 `'skipped'` 状态

**选择 B**。

**理由**：
1. **语义清晰**：`'skipped'` 与"未做过"（pending）、"已完成"（done）三态互斥，TopicTransitionView 列表能展示区分图标（如跳过=灰色虚线圈）
2. **allDone 逻辑明确**：`allDone = 所有 module 状态 ∈ {'done', 'skipped'}`。skipped 计为"已处理"，不阻塞主题完成判定
3. **可回看补做**：skipped 状态的 module 可在 TopicTransitionView 点击重新进入（状态回退为 in_progress），区别于 pending 的"从未进入"
4. **TopicProgress 快照**：`exitSession` 写 `completedModuleIds` 时，skipped module 不写入（不算完成），但 currentIndex 推进——下次 resume 时 skipped 仍标记为 skipped

#### D-2：F18 用 middleware.ts + env 密码（非 vercel.json Basic Auth）

**选项**：A) `vercel.json` headers Basic Auth / B) `src/middleware.ts` + env 密码 + cookie

**选择 B**。

**理由**：
1. **dev 也生效**：Basic Auth 仅 Vercel 生产环境生效，本地 `bun run dev` 无保护；middleware 在所有环境生效
2. **UX 更好**：middleware 可渲染自定义密码输入页（而非浏览器原生 Basic Auth 丑陋弹窗），与项目暗色主题一致
3. **灵活**：env `STUDIO_PASSWORD` 未设置时自动放行（开发便利），设置后强制校验
4. **实现**：middleware 匹配 `/studio/:path*`，检查 `studio-auth` cookie；无 cookie 重定向 `/studio/login`；login 页 POST 校验密码后 set-cookie（httpOnly + sameSite=strict）

#### D-3：F42 编辑题目扩展现有 AnswerCorrector（非新建组件）

**选项**：A) 新建 `QuizEditor` 组件 / B) 扩展现有 `AnswerCorrector` 加编辑模式

**选择 B**（扩展现有，组件名可保留 AnswerCorrector 或重命名为 QuizEditor）。

**理由**：
1. **复用分发逻辑**：AnswerCorrector 已按 `interactionType` 分发 ChoiceEditor/SortingEditor/FillBlankEditor，编辑正文只需在这些 editor 内加"题干/选项/解析"输入区
2. **复用持久化链路**：`correctQuizAnswer` + `updateQuizInModule` 已支持任意 `Partial<Quiz>` patch，编辑正文只是 patch 字段更多（stem / options / explanation / distractors）
3. **避免双入口**：新建组件会产生"修正答案走 A、编辑正文走 B"两个入口，用户困惑。合并为单入口"编辑此题"更自然

**编辑范围**（F42 在 F40 基础上扩展的可编辑字段）：

| 题型 | F40 已可改 | F42 新增可改 |
| --- | --- | --- |
| choice | `answer` | `stem` / `options[]` / `explanation` / `distractors[]` |
| sorting | `answer` + `options[]`（顺序） | `stem` / `explanation` |
| fill_blank | `answer` + `acceptableAnswers[]` | `stem` / `explanation` / `answerHint` |

#### D-4：F40 扩展 Challenge 修正是纯接线

**现状**：AnswerCorrector 已支持 3 题型；ChallengeView 用同结构 Quiz（`module.challengeQuizzes[]`），但未渲染 AnswerCorrector。

**做法**：ChallengeView 按 ConceptView 的模式接入 `correctQuizAnswer` + `reevaluateLastAttempt` + FeedbackPanel 的 `canCorrect/onCorrectAnswer` props。

**注意**：ChallengeView 的 retry 逻辑（synthesizes fake Concept for `/api/regenerate`）不改；仅加修正/忽略入口。Challenge quiz 的 `slotId` 构造需与 attempts-store 约定一致（`challenge:{quizId}`）。

#### D-5：F43 忽略题管理页用"题库页内嵌分区"（非独立路由）

**选项**：A) 独立路由 `/learn/ignored` / B) 题库页（`/learn/library`）内嵌"已忽略题目"分区

**选择 B**。

**理由**：
1. **信息架构一致**：忽略的题属于某个 Module，在题库页按 module 聚合展示最自然（"毛概工作方法 - 2 题已忽略"）
2. **减少路由膨胀**：项目已 9 个 learn 页，再加一个边际功能路由不经济
3. **批量操作**：分区支持"全选恢复"/"按 module 恢复"，复用现有 library 组件模式

#### P1.3 F18 /studio 访问保护

| 项 | 说明 |
| --- | --- |
| 文件 | `src/middleware.ts`（新建）、`src/app/studio/login/page.tsx`（新建）、`.env.example`（加 STUDIO_PASSWORD 说明） |

**middleware.ts**：

```typescript
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

const STUDIO_PATH = '/studio'
const COOKIE_NAME = 'studio-auth'

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl

  // 仅匹配 /studio 及其子路径（排除 /studio/login 自身）
  if (pathname === '/studio/login') return NextResponse.next()
  if (!pathname.startsWith(STUDIO_PATH)) return NextResponse.next()

  // env 未设置密码 → 放行（开发便利）
  const password = process.env.STUDIO_PASSWORD
  if (!password) return NextResponse.next()

  // 校验 cookie
  const authed = request.cookies.get(COOKIE_NAME)?.value === 'ok'
  if (!authed) {
    const loginUrl = new URL('/studio/login', request.url)
    return NextResponse.redirect(loginUrl)
  }
  return NextResponse.next()
}

export const config = {
  matcher: ['/studio/:path*'],
}
```

**login 页**：暗色主题密码输入表单，POST 校验 `STUDIO_PASSWORD`，通过则 `set-cookie`（httpOnly + sameSite=strict + maxAge=7d），重定向 `/studio`。

**验收**：
- `STUDIO_PASSWORD` 未设置时 `/studio` 直接可访问
- 设置后未认证访问 `/studio` 重定向 login
- login 输入正确密码后进入，cookie 7 天有效
- middleware 不影响 `/studio` 以外路由

#### P3.1 F24 过渡页"跳过此题库"

| 项 | 说明 |
| --- | --- |
| 文件 | `src/types/domain.ts`（ModuleTopicStatus 加 skipped）、`src/lib/state/topic-session-store.ts`（加 skipCurrentModule）、`src/components/learn/TopicTransitionView.tsx`（加按钮 + skipped 展示） |

**域模型扩展**：

```typescript
// src/types/domain.ts
export type ModuleTopicStatus = 'pending' | 'in_progress' | 'done' | 'skipped'
```

**topic-session-store 新增 action**：

```typescript
interface TopicSessionStoreState {
  // ... 现有

  /**
   * 跳过当前 module（F24）。推进 currentIndex，标记当前为 'skipped'。
   * skipped 不写入 TopicProgress.completedModuleIds（不算完成）。
   * 返回下一个 moduleId（若有）。
   */
  skipCurrentModule: () => string | null
}
```

实现要点：
- 当前 module 状态置 `'skipped'`
- `currentIndex + 1`，下一个 module 置 `'in_progress'`
- `allDone` 判定：`所有 module ∈ {'done', 'skipped'}`
- `exitSession` 写 TopicProgress 时 skipped module 不写入 completedModuleIds

**TopicTransitionView 改动**：

1. 模块列表中 skipped 状态显示灰色虚线图标 + "已跳过"标签
2. skipped module 可点击重新进入（状态回退 in_progress）
3. 底部加"跳过此题库"按钮（inline confirm：`确认跳过？可稍后回来 [确认] [取消]`）
4. 跳过后自动推进到下一个 module

**边界处理**：
- 跳过最后一个 module → allDone = true（因 skipped 计为已处理）→ 展示主题完成页
- 全部跳过 → 主题完成，但聚合 mastery（F23）极低（正确行为，反映用户没真正学）

**验收**：
- TopicTransitionView 有"跳过此题库"按钮
- 跳过后当前 module 标记 skipped，推进到下一个
- skipped module 在列表有区分展示
- 跳过最后一个 module 触发主题完成
- skipped module 可点击重新进入
- 退出主题后 resume，skipped 标记保留

## 5. 风险登记簿

### 5.1 高优先级

| 编号 | 风险 | 影响 | 缓解 |
| --- | --- | --- | --- |
| R-H1 | **E2E 回归发现大量失败** | Phase 1 工期膨胀 | 逐个修复，回归属 v1.1.0/v1.1.1 改动副作用；若 >10 个失败需评估是否扩大 Phase 1 范围 |
| R-H2 | **F42 编辑正文破坏 Quiz 结构不变量** | 序列/mastery/retry 逻辑异常 | 编辑后跑现有 639 单测 + 新增编辑单测；options 增删需重验 adaptive-sequencer |
| R-H3 | **F24 skipped 状态影响 TopicProgress resume** | resume 后状态错乱 | TopicProgress 仅存 completedModuleIds（不含 skipped）；resume 时从 TopicSession 重读 skipped 标记；单测覆盖 skip→exit→resume 链路 |

### 5.2 中优先级

| 编号 | 风险 | 影响 | 缓解 |
| --- | --- | --- | --- |
| R-M1 | **F18 middleware 影响其他路由性能** | 全局延迟 | matcher 限定 `/studio/:path*`，middleware 仅对 studio 生效；其余路由不经过 |
| R-M2 | **Challenge quiz slotId 约定不一致** | F40 扩展修正后重评错位 | 接入前审计 ChallengeView 现有 slotId 构造，对齐 attempts-store 键约定 |
| R-M3 | **OG PNG 渲染工具链缺失** | 阻塞 P1.2 | SVG→PNG 可用 `sharp`（Node）或在线工具；不阻塞其他项 |
| R-M4 | **F42 编辑 options 数量变化影响 retry** | retry 生成的 quiz 与编辑后不匹配 | retry 走 `/api/regenerate` 生成全新 quiz，不依赖原 options 数量；验证 retry 后编辑态重置 |

### 5.3 低优先级

| 编号 | 风险 | 缓解 |
| --- | --- | --- |
| R-L1 | middleware cookie 被 CSRF | sameSite=strict + 仅 studio 路径，风险极低 |
| R-L2 | IgnoredQuizSection 扫描全库性能 | user-origin Module 数量有限（非 showcase 全量）；按需懒加载 |
