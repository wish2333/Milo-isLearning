## 2. 架构决策记录

### 2.1 D-1：F04 checkpoint 粒度 = 按 stage（非按 concept）

选择 stage 4（module 组装完成）和 stage 6.5（challenge 完成）两个粗粒度 checkpoint，而非 per-concept 细粒度。理由：quiz stage 使用 concurrency=3 并行 + 就地变异 partialModule，per-concept checkpoint 需重构并行逻辑，风险远大于收益。

### 2.2 D-2：F08 token 追踪通过 runAgent() 返回值变更

选择 `runAgent<T>()` 返回 `Promise<{ data: T; usage: TokenUsage }>` 而非回调或包装器。仅 12 个调用点（全部在 pipeline.ts + 3 个 API route + 1 个 eval script），改动面可控。

### 2.3 D-3：F11 双轨策略 = production SQLite / showcase LocalStorage

Production 模式事件写 SQLite events 表；showcase 模式继续 LocalStorage 落盘。Showcase 零后端原则不变。

### 2.4 D-4：F07 重命名不改 moduleId

仅更新 Module.title 字段。moduleId 被 7 个关联 key 引用（attempts/mastery/progress/feynman/quality 等），改 ID 等于局部迁移，风险远大于收益。所有 UI 从存储的 Module 对象实时读取 .title（约 20 处），改值即自动传播。

### 2.5 D-5：F22 主题进度恢复用独立 storage key

新增 `alc:topic-progress:{topicId}` 存储已完成模块快照，不复用 TopicSession（运行时会话状态）。独立 key 可被 TopicCard 等组件读取展示"上次进度"。

### 2.6 PB.2 Pipeline checkpoint 回调解耦

Pipeline 不直接导入 `server-only` 和 persistence 层。通过 CompileOptions.writeCheckpoint 回调注入，保持 pipeline 模块纯粹。checkpoint 用 string key（如 `'module-concepts'`）而非 CompileStage 枚举，支持复合 checkpoint。

### 2.7 PB.2 Resume stage 跳过策略

每个 stage 开头检查 `isStageCompleted(stage)`：若 true 则从 checkpointData 恢复 artifact，跳过 runAgent 调用。Stage 4 resume 有特殊处理——yield 一个带"从断点恢复"消息的 progress event。

## 4. 已知限制

### 4.1 Schema 版本 meta 不更新

`INSERT OR IGNORE` 保证幂等，但 v1/v2 数据库升级到 v3 后 `meta.schema_version` 值不更新（仍为创建时的值）。events/compile_session/compile_staging 表通过 `CREATE TABLE IF NOT EXISTS` 自动创建，功能正常。如需精确版本检测，可查 `sqlite_master` 表存在性。

### 4.2 OG 图片为 SVG 格式

`public/og-image.svg` 是 SVG 格式。部分社交平台（Facebook/LinkedIn）不支持 SVG OG 图片。生产部署前应转换为 PNG（1200x630, <300KB）。

### 4.3 字体分包需手动执行

`scripts/split-fonts.ts` 已就位但需手动运行，且需要 Source Han Serif SC 原始字体文件（项目仓库不含）。首次部署前需执行分包构建。

### 4.4 F04 checkpoint 仅 production 模式

Showcase 模式编译是 mock 回放，无 LLM 调用，不需要 checkpoint。所有 checkpoint 逻辑通过 `sessionId` 存在性守卫，showcase 模式自动跳过。

### 4.5 E2E 基线待回归对比

v1.0.1 建立的 51 tests E2E 基线尚未在 v1.1.0 上重跑。Phase B 的 pipeline 改造（PB.2）和 storage schema 升级可能影响 showcase 回归。建议合入 main 前运行 `bun run e2e`。

### 4.6 PC.2 英文模板仅 PoC 2 个

只翻译了 import.md + quiz-batch.md 两个高优先级模板。其余 8 个模板仍为中文。完整英文化需后续迭代。
