## 2. 架构总览

### 2.1 V1.0.0 基线回顾

V1.0.0 已交付的 production 模式基础设施是 V1.1.0 的前置条件：

- **SQLite 持久化层**：4 表 schema（`kv` / `meta` / `migration_session` / `migration_staging`）+ WAL 模式 + 单例工厂
- **客户端写队列**：FIFO + per-key 合并 + 3 档退避 + FlushManager
- **API 路由**：`/api/data/*`（CRUD + bulk/clear/export/flush/import/restore/status）+ `/api/migrate/*`（5 路由）
- **BackupPackage**：V1 Zod schema + 长度前缀 SHA-256 + VACUUM INTO 快照

### 2.2 V1.1.0 架构变更概览

| 变更域 | 新增 | 修改 |
| --- | --- | --- |
| SQLite schema | `events` 表（F11）、`compile_session` + `compile_staging` 表（F04） | schema 版本 1 → 2 |
| API 路由 | `/api/events`（F11）、`/api/compile/{session,checkpoint,resume}`（F04） | `/api/compile/route.ts` 加 AbortSignal |
| 编译管线 | checkpoint 写入点（F04）、token 累计器（F08）、mapper fix 计数器（F33） | `runAgent()` 返回类型变更（F08） |
| 埋点 | `app_mode` 自动注入（F20）、env fallback 事件（D5） | `analytics.ts` track() 签名 |
| 域模型 | `TopicMastery` 类型（F23）、`TopicProgress` 快照（F22） | `Module` 不变（F07 改值不改结构） |
| Prompt | 版本切换机制（F30）、英文模板（F31）、录制回放（F32） | `loader.ts` 加载逻辑 |
| 前端资源 | 字体分包（F10）、OG 图片（D2）、a11y 修复（D1） | `globals.css`、`layout.tsx` |

### 2.3 核心设计决策

#### D-1：F04 checkpoint 粒度 = 按 stage（非按 concept）

**选项**：按 concept（quiz 阶段细粒度） / 按 stage（8 阶段粗粒度）

**选择**：按 stage。在 stage 4（`partialModule` 组装完成）和 stage 6.5（challenge 完成）设两个 checkpoint。

**理由**：
1. Quiz 阶段使用 `concurrency=3` 并行 + 就地变异 `partialModule.concepts[*].quizSeries.quizzes`（pipeline.ts:722-737），按 concept checkpoint 需要重构并行逻辑，风险高
2. Stage 1-3 的产物是简单字符串/数组（normalizedText / chunks / conceptsRaw），重新执行成本低（<10s），不值得 checkpoint
3. Stage 4 之后的 stage（mission/quiz/challenge/feynman）都依赖 `partialModule`，checkpoint 此对象可覆盖所有后续 stage 的恢复需求
4. migration_staging 模式（session + staging 两表）已证明可行，直接复用

#### D-2：F08 token 追踪通过 `runAgent()` 返回值变更实现

**选项**：A) `runAgent()` 返回 `{ data, usage }` / B) onUsage 回调 / C) 外部包装器

**选择**：A。`runAgent<T>()` 返回类型从 `Promise<T>` 改为 `Promise<{ data: T; usage: TokenUsage }>`。

**理由**：
1. 项目红线禁止 `as any`，类型安全要求显式传递
2. 仅 8 个 stage 调用点（pipeline.ts 内），改动面可控
3. ChatResponse.usage 已由 provider 层返回（providers/types.ts:68-72），只是被 `_runner.ts:138` 丢弃
4. 回调方式容易遗漏；包装器方式引入间接层

#### D-3：F11 双轨策略 = production SQLite / showcase LocalStorage

**选择**：production 模式事件写 SQLite `events` 表；showcase 模式继续 LocalStorage 落盘（现有行为）。

**理由**：
1. Showcase 模式零后端原则不变
2. Production 模式已有 `/api/data/*` 基础设施，加 `/api/events` 仅需 1 路由 + 1 表
3. Showcase 事件可通过 BackupPackage 手动导出（已有机制）

#### D-4：F07 重命名不改 moduleId

**选择**：`renameModule(repo, moduleId, newTitle)` 仅更新 `Module.title` 字段，不改 `moduleId`、不改 storage key。

**理由**：
1. `moduleId` 被 attempts / mastery / progress / feynman / quality 等 7 个关联 key 引用（见 v1.0.0-build-plan.md §5 Phase 5 关联数据完整性）
2. 改 moduleId 需要级联更新所有关联 key，等于一次"局部迁移"，风险远大于收益
3. 所有 UI 读 `module.title` 是从存储的 Module 对象实时读取（共 ~20 处），改值即自动传播

#### D-5：F22 主题进度恢复用独立 storage key，不复用 TopicSession

**选择**：新增 `alc:topic-progress:{topicId}` 存储已完成模块快照，`exitSession` 时写入，`startSession` 时检查并合并。

**理由**：
1. TopicSession 是运行时会话状态（含 currentIndex 指针），退出后语义不同
2. 独立 key 可被 TopicCard 等组件读取展示"上次进度"，不需激活 session
3. 避免持久化临时性的 currentIndex（用户可能下次想从第一个未完成模块开始）

### 2.4 约束红线（继承 v1.0.0 + 新增）

1. **showcase 用户行为和数据语义零回归**（继承）
2. **不引入 `as any` / `@ts-ignore`**（继承）
3. **不裸用 `localStorage`**——唯一例外 `legacy-local-storage-scanner.ts`（继承）
4. **不生成 emoji**（继承 CLAUDE.md）
5. **中文优先**（继承）
6. **server-only / client-only 强制隔离**（继承）
7. **F04 仅 production 模式生效**——showcase 模式编译是 mock 回放，无需 checkpoint
8. **F11 双轨不交叉**——production 事件不写 LS，showcase 事件不调 `/api/events`
9. **Schema 升级必须向后兼容**——v1 数据库升级到 v2 时自动执行 DDL，不破坏现有数据

```typescript
// analytics.ts 改造后的 flush 逻辑
async function flush(): Promise<void> {
  if (buffer.length === 0) return
  const batch = buffer.splice(0)
  if (isProductionMode) {
    try {
      await fetch('/api/events', { method: 'POST', body: JSON.stringify({ events: batch }) })
    } catch { /* 失败回填 buffer，下次重试 */ }
  } else {
    // showcase: 写 LocalStorage（现有逻辑保留）
    persistToLocal('alc:events', batch)
  }
}

// track() 自动注入 app_mode
function track(name: string, props?: EventProps): void {
  buffer.push({ name, props: { ...props, app_mode: APP_MODE }, timestamp: Date.now() })
}
```

**成本估算**：

```typescript
// pricing 表（可配置，初版硬编码）
const PRICING_USD_PER_1M_TOKENS: Record<string, { input: number; output: number }> = {
  deepseek: { input: 0.14, output: 0.28 },
  glm: { input: 0.50, output: 0.50 },
  'openai-compat': { input: 1.00, output: 3.00 },  // 保守估计
}

function estimateCost(usage: TokenUsage, provider: string): EstimatedCost {
  const pricing = PRICING_USD_PER_1M_TOKENS[provider] ?? { input: 1.00, output: 3.00 }
  return {
    inputCost: (usage.promptTokens / 1_000_000) * pricing.input,
    outputCost: (usage.completionTokens / 1_000_000) * pricing.output,
    totalCost: 0,  // computed
    currency: 'USD',
  }
}
```

**验收**：
- 编译完成后 QualitySummary 显示 "输入 X / 输出 Y tokens / 估算 $Z"
- `compile_complete` 事件携带 `totalTokens`
- 单测：estimateCost 给定 usage + provider 返回正确数值

#### PB.2 F04 Pipeline 改造 + SSE route AbortSignal

| 项 | 说明 |
| --- | --- |
| 文件 | `src/lib/compiler/pipeline/pipeline.ts`、`src/app/api/compile/route.ts` |
| 现状 | `compileMarkdown()` 无 sessionId 参数；route.ts 无 AbortSignal，客户端断开后 pipeline 继续跑 LLM 调用 |

**Pipeline 改造**：

1. `compileMarkdown()` 新增可选参数 `options?: { sessionId?: string; resumeFrom?: CompileStage }`
2. 每个 stage 完成后（`yield* runStage()` 返回后），若 `sessionId` 存在则调 `saveCheckpoint()`
3. 若 `resumeFrom` 存在，跳过已完成的 stage，从 staging 表加载 artifact 注入局部变量
4. Checkpoint 时机：stage 4（module 组装完成）+ stage 6.5（challenge 完成）— 两个粗粒度 checkpoint

**Checkpoint 策略（按 D-1 决策）**：

| Checkpoint 点 | 捕获的 artifact | 覆盖的恢复范围 |
| --- | --- | --- |
| After stage 4 (module) | `partialModule` + `concepts` + `conceptOrder` + `sourceId` | stage 5-7 可恢复 |
| After stage 6.5 (challenge) | 更新后的 `partialModule`（含 quiz + challenge） | stage 7 可恢复 |

> Stage 1-3 不 checkpoint（重新执行 <10s，成本低）。
> Quiz stage（6）内部不 checkpoint（并行 + 变异，风险高），stage 6.5 完成后统一捕获。

**Route.ts AbortSignal**：

```typescript
// route.ts 改造
export async function POST(req: NextRequest) {
  const signal = req.signal  // AbortSignal
  const stream = new ReadableStream({
    async start(controller) {
      const generator = compileMarkdown(...)
      try {
        for await (const event of generator) {
          if (signal.aborted) {
            generator.return(undefined)  // 触发 generator cleanup
            break
          }
          controller.enqueue(encoder.encode(formatSSE(event)))
        }
      } finally {
        controller.close()
      }
    }
  })
  return new Response(stream, { headers: SSE_HEADERS })
}
```

```typescript
// mastery.ts 或新文件 topic-mastery.ts
export function computeTopicMastery(
  topic: Topic,
  modules: Module[],
  attemptsByModule: Record<string, Record<string, AttemptRecord[]>>,
  feynmanAttempts: Record<string, FeynmanAttempt>,
): TopicMastery {
  const moduleMasteries = modules.map(m => {
    const mastery = computeMastery(m, attemptsByModule[m.id] ?? {}, feynmanAttempts[m.id])
    const weight = m.concepts.reduce((sum, c) => sum + c.quizSeries.quizzes.length, 0)
    return { moduleId: m.id, moduleTitle: m.title, mastery, weight }
  })
  const totalWeight = moduleMasteries.reduce((s, mm) => s + mm.weight, 0)
  const aggregateMastery = totalWeight > 0
    ? Math.round(moduleMasteries.reduce((s, mm) => s + mm.mastery.moduleCompletion * mm.weight, 0) / totalWeight)
    : 0
  return { topicId: topic.id, aggregateMastery, moduleMasteries, totalQuizzes: totalWeight, completedModules: moduleMasteries.filter(mm => mm.mastery.moduleCompletion === 100).length }
}
```

#### PD.4 D4 openai-compat baseURL 运行时检测

| 项 | 说明 |
| --- |---|
| 来源 | V1.0.0-Review §6.3 |
| 文件 | `src/lib/providers/env-fallback.ts`、`src/lib/providers/openai-compat-provider.ts` |
| 现状 | `env-fallback.ts:15` openai-compat 默认 baseURL 为空字符串 `''`；`openai-compat-provider.ts:30-32` 因 `''` falsy 直接抛错 |

**改造方案（两选一）**：

**方案 A（保守，推荐）**：`env-fallback.ts` openai-compat 无 baseURL 时返回 `null`（不构造 LLMConfig），让 fallback 失败而非崩溃。

```typescript
// env-fallback.ts
if (provider === 'openai-compat' && !baseURL) {
  return null  // 无法 fallback，返回 null
}
```

**方案 B（智能）**：`openai-compat-provider.ts` 加 baseURL 启发式检测。

```typescript
// openai-compat-provider.ts
function detectBaseURL(apiKey: string, model: string): string | null {
  if (apiKey.startsWith('sk-') && model.startsWith('gpt-')) return 'https://api.openai.com/v1'
  if (apiKey.startsWith('anthropic-')) return 'https://api.anthropic.com/v1'
  return null
}
```

**选择 A**：保守方案，避免误检测导致请求发到错误端点。

**验收**：
- Showcase 模式下 openai-compat 无 baseURL → env fallback 返回 null → 清晰错误提示
- 不再因空字符串 baseURL 崩溃
- 单测：env-fallback openai-compat 无 baseURL 返回 null
