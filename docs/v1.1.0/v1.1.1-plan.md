## 0. 阅读指南

### 0.1 背景：为什么需要这个版本

LLM 生成的题库存在一个高频质量问题——**题目本身和解析都很好，但"标准答案"标错了**。典型表现：

- 选择题：4 个选项里正确答案明明是 B，LLM 却把 A 标成了 `quiz.answer`
- 排序题：正确顺序被打乱后 LLM 指认了错误的基准序列
- 填空题：`quiz.answer` 与 `explanation` 描述的要点不一致

这类错误的代价很高：用户明明答对了却被判错，或者照着错误答案"学"了错误知识。v1.0.0-v1.1.0 的自动质量校验（Mapper fix stats F33、autoFixQuizBatch）能修一部分结构性问题（如 answer 不在 options 里），但**无法判断语义正确性**——只有人在作答时才能发现"这题答案不对"。

因此需要一个**人工介入的闭环**：用户在作答结果页发现答案标错时，能直接修正答案或忽略该题，而不必删掉整个题库重编。

## 2. 架构总览

### 2.1 现状回顾：作答反馈链路

当前作答反馈链路（来自代码审计）：

```
用户作答 (ChoiceQuiz/SortingQuiz/FillBlankQuiz)
   ↓ onAnswer(userAnswer)
ConceptView.handleAnswer()
   ↓ evaluateAnswerAsync(quiz, userAnswer, provider)   ← 判分
   ↓ addAttempt(attempt)                                 ← 写入 attempts-store（按 slotId 分组）
   ↓ setFeedback(result); setPhase('feedback')           ← 进入反馈态
   ↓ 渲染 <FeedbackPanel feedback={...} />               ← 展示解析 + 蒙对标注
   ↓ 用户点"继续" → advance()                            ← 状态机推进
```

关键数据流：

- `Quiz` 存储在 `Module.concepts[].quizSeries.quizzes[]`（及 `Module.challengeQuizzes[]`），通过 `module-library` 持久化，`module-store.currentModule` 持有内存副本
- `AttemptRecord` 按 `originalQuizId`（槽位 id）存入 `attempts-store`
- `computeMastery()` 是纯函数，从 `module + attemptsBySlot + feynmanAttempt` 计算 Mastery
- `FeedbackPanel` 在 ConceptView（学习中）和两个 review 页（错题重刷）共 3 处渲染

### 2.3 核心设计决策

#### D-1：F41 用"软删除"（ignored flag）而非硬删除

**选项**：A) 硬删除（从 `quizzes[]` 数组移除）/ B) 软删除（`quiz.ignored = true`）

**选择 B**。

**理由**：
1. **可逆**：用户可能误判，软删除可一键恢复；硬删除不可逆
2. **数据完整性**：`moduleId` / `quizId` 被 attempts、mastery、progress、feynman 等多处引用。硬删除会导致孤儿引用（与 v1.0.1 修复的 module 孤儿问题同类）
3. **审计性**：保留原题数据，便于后续分析"哪些题容易被忽略"（Prompt 质量反馈）
4. **语义清晰**：ignored 题"不参与掌握度计算、不在序列中展示"，与"物理不存在"区分明确

#### D-2：F40 修正答案后触发"重评最后一答 + 掌握度重算"

**选择**：修正答案 → 用新答案重新评估该 slot 的最后一条 AttemptRecord → 更新其 score/nextAction → mastery 随 attempts 变化自动重算。

**理由**：
1. mastery 计算是纯函数（`computeMastery`），依赖 `attemptsBySlot`。只要 attempt 的 score 更新，mastery 即正确
2. 只重评"最后一条"而非全部——因为掌握度只看首次答对率（`attemptVersion === 0`），重评首条即可纠正判定
3. 不删除旧 attempt、不新增 attempt——原地更新 score 字段，保留作答历史时间线

#### D-3：修正范围 = 仅 `answer` 字段（及其衍生字段）

本版本**只允许修正答案**，不改正文/选项文本/解析。具体：

| 题型 | 可修正字段 | 不可修正字段 |
| --- | --- | --- |
| choice | `answer`（选哪个选项） | stem / options[] / distractors[] / explanation |
| sorting | `answer` + `options[]`（正确顺序） | stem / explanation |
| fill_blank | `answer` + `acceptableAnswers[]` | stem / explanation / answerHint |

**理由**：用户痛点明确是"答案标错"，正文和解析是好的。扩大编辑范围会引入内容编辑器（富文本、校验），超出本版本"人工纠错闭环"的定位。若用户觉得正文也要改，可在后续版本扩展。

#### D-4：showcase origin 题库不可修正/忽略

**选择**：`module.origin === 'showcase'` 时，FeedbackPanel 不显示修正/忽略按钮（与 F07 重命名对 showcase 的保护一致）。

**理由**：
1. 展示题库是精选内容，用户级修正会污染一致性（不同用户看到不同答案）
2. showcase 模式是 demo 定位，数据正确性由作者保证
3. 若展示题库确实有错，应走"修复源文件 → 重新发布"流程，而非运行时改

> **例外考虑**：若内测反馈强烈要求"展示题库也能个人级修正（仅本地生效）"，可在 v1.1.2 评估。本版本先禁用。

#### D-5：二次确认用 inline confirm（非 modal dialog）

**选择**：点击"修正答案"/"忽略此题"后，按钮原地变为"确认修正？[确认] [取消]"，不弹模态框。

**理由**：
1. 与现有"蒙对标注"的交互风格一致（FeedbackPanel 内的轻量操作）
2. 模态框打断学习心流；inline confirm 保持上下文
3. 两步操作（点按钮 → 点确认）已满足"高危操作需二次确认"的要求

#### D-6：修正答案的 UI 形态按题型分发

| 题型 | 修正 UI |
| --- | --- |
| choice | 展示所有选项，用户点选正确的那个（类似再答一次，但目的是"标记正解"） |
| sorting | 展示可拖拽列表，用户排成正确顺序 |
| fill_blank | 展示文本输入框，用户填写正确答案（可填多个 acceptableAnswers） |

**理由**：复用现有 QuizRenderer 的交互形态，用户学习成本零。但需要一个新的"修正模式"渲染态（只读选项 + 标记正解，不触发 onAnswer）。

### 2.4 约束红线（继承 + 新增）

继承 `v1.1.0-plan.md` §2.4 全部红线，新增：

12. **F40/F41 不引入 `as any`**（继承 #2）——Quiz 类型扩展用可选字段 `ignored?: boolean`，向后兼容
13. **F40 重评不新增 AttemptRecord**——原地更新最后一条的 score 字段
14. **F41 ignored 题的 attempt 保留**——不删除历史作答，仅在 mastery 计算时排除
15. **二次确认不可跳过**——两个高危操作必须经过"点按钮 → 点确认"两步，无快捷键绕过
16. **修正/忽略操作仅在 feedback phase 可用**——未作答时不显示（避免未看解析就改答案）

#### P1.1 域模型与持久化层

| 项 | 说明 |
| --- | --- |
| 文件 | `src/lib/persistence/module-library.ts`（新增函数）、`src/types/domain.ts`（无结构变更） |

**新增函数** `module-library.ts`：

```typescript
/**
 * 更新 Module 内某道 Quiz 的字段（不可变更新）。
 * 用于 F40 修正答案、F41 标记忽略。
 *
 * @param moduleId 目标 Module
 * @param quizId 目标 Quiz
 * @param patch 要更新的字段（如 { answer: '新答案' } 或 { ignored: true }）
 * @throws 若 module 或 quiz 不存在
 */
export function updateQuizInModule(
  repo: StorageRepository,
  moduleId: string,
  quizId: string,
  patch: Partial<Pick<Quiz, 'answer' | 'options' | 'acceptableAnswers' | 'ignored'>>,
): Module
```

实现要点：
- 从存储加载 Module → 深度查找 quiz（遍历 concepts[].quizSeries.quizzes[] + challengeQuizzes[]）
- 不可变更新（spread 重建路径）→ 写回存储 → 返回更新后的 Module
- 不改 quizId（继承 D-4 of v1.1.0：不改 ID 避免级联迁移）

**验收**：
- `updateQuizInModule` round-trip 单测（改 answer → 重载 → 新值在）
- 改一个 quiz 不影响同 module 其他 quiz
- 不存在的 moduleId/quizId 抛明确错误

#### P1.2 attempts-store 重评能力

| 项 | 说明 |
| --- | --- |
| 文件 | `src/lib/state/attempts-store.ts`（新增 action） |

**新增 action**：

```typescript
interface AttemptsStoreState {
  // ... 现有

  /**
   * 用修正后的 quiz 重新评估某 slot 的最后一条作答，原地更新其 score。
   * 用于 F40 答案修正后纠正历史判定。
   */
  reevaluateLastAttempt: (
    slotId: string,
    correctedQuiz: Quiz,
    provider?: LLMProvider | null,
  ) => Promise<void>
}
```

实现要点：
- 取 `attemptsBySlot[slotId]` 最后一条
- 调 `evaluateAnswerAsync(correctedQuiz, lastAttempt.userAnswer, provider)` 重评
- 原地更新该 attempt 的 `score` / `gaps` / `nextAction`（保留 userAnswer / timestamp / guessed 不变）
- async（fill_blank 可能触发语义判分 LLM 调用）

**验收**：
- 单测：标错答案 → 用户答对被判错 → 修正答案 → 重评后 score 100
- 单测：重评不新增 attempt、不改 timestamp
- 重评后 progress-store 的 mastery 订阅自动触发（因 attemptsBySlot 引用变化）

改动：
- Concept 内 quiz 序列：遇到 `ignored === true` 的题自动跳到下一道（不展示、不计数）
- 重刷队列构建（`buildAdaptiveQueue` / `buildTopicReviewQueue`）：排除 ignored 题

**边界处理**：
- 某 concept 所有题都被忽略 → 该 concept 视为已完成（避免卡死）
- 整个 module 所有题被忽略 → moduleCompletion = 100%（无可学内容）

**验收**：
- 单测：3 题中忽略第 2 题 → 学习序列为 1→3，不展示第 2 题
- 单测：重刷队列不含 ignored 题

---

#### P2.4 忽略 UI 接入（复用 Phase 1）

| 项 | 说明 |
| --- | --- |
| 文件 | `src/components/quiz/FeedbackPanel.tsx`（已在 P1.4 加 onIgnoreQuiz prop） |

FeedbackPanel 的"忽略此题"按钮（P1.4 已建）点击流程：
1. inline confirm：`确认忽略此题？将不计入掌握度 [确认] [取消]`（destructive 红色样式）
2. 确认 → 调 `onIgnoreQuiz` → 父组件调 `module-store.correctQuizAnswer(quiz.id, { ignored: true })`
3. UI 反馈："已忽略" + 自动 advance 到下一题（因当前题不再属于序列）

**恢复机制**：忽略后若用户反悔——题库列表页或重刷页可展示"已忽略的题"并提供"恢复"入口（本版本用最小方案：FeedbackPanel 内若 `quiz.ignored === true` 显示"已忽略 [撤销忽略]"）。撤销即 `correctQuizAnswer(quiz.id, { ignored: false })`。

**验收**：
- 忽略后当前题消失，自动进下一题
- 忽略后 mastery 重算（分母变小）
- 撤销忽略后题目回到序列
