## 3. 架构决策记录

### 3.1 P1.2 M4 progress-store subscribe 是 ACTIVE 的（非注释代码）

**build-review §4.3 原描述**："`progress-store.ts:347-366` 有 LS+SQLite 双写 subscribe 代码（注释保留），目前注释掉未启用"

**实际代码**（commit `2504cb4`，line 381-400）：subscribe 是 **export 的活跃代码**，非注释。`_unsubscribeProgressSync` 在模块加载时立即注册，持续监听 progress-store 变更并同步到 per-module key `alc:progress:{moduleId}`。

**影响**：`listStoredModules` 从 per-module key 读取 `updatedAt` / `completed`，如果删除 subscribe，题库列表的进度状态将永远过时。

**决策**：保留代码不变。修正 build-review 的错误描述。

### 3.2 P1.1 isSameOrigin 反代加固方案

**原实现**：`origin === http://${host} || origin === https://${host}` — 硬猜协议，且仅读 Host header。

**新实现**：`new URL(origin).host === resolveForwardedHost(req)` — 解析 Origin 的 host（含端口），与 resolved host（优先 X-Forwarded-Host → Forwarded → Host）比较。不关心协议，更安全更正确。
