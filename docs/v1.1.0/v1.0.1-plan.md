### 2.3 核心设计决策

#### D-1：E2E webServer 改为双服务器 + 环境变量驱动

**现状**：`playwright.config.ts` 仅定义 1 个 webServer（`bun run dev` @3000），但：
- showcase 项目 baseURL 是 `http://localhost:3001`（无 webServer 启动）
- `e2e/storage-layer.spec.ts` 8 个测试需要 `E2E_PRODUCTION_MODE=true` + production 模式 dev server
- 默认 `bun run dev` 启动的是 showcase 模式（v1.0.0 后默认），api-data 测试可能假设 production

**选择**：改为双 webServer + env 注入：

```typescript
// playwright.config.ts 改造后
const RUN_PRODUCTION = process.env.E2E_PRODUCTION_MODE === 'true'

webServer: [
  {
    command: RUN_PRODUCTION
      ? 'bun run dev'  // 配合 .env.e2e-production 设 NEXT_PUBLIC_APP_MODE=production + ALC_STORAGE_BACKEND=sqlite
      : 'bun run dev',
    env: RUN_PRODUCTION
      ? { NEXT_PUBLIC_APP_MODE: 'production', ALC_STORAGE_BACKEND: 'sqlite' }
      : undefined,
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
    timeout: 120000,
  },
  {
    command: 'bun run dev',  // showcase @3001
    env: { PORT: '3001' },  // Next.js 服从 PORT env
    url: 'http://localhost:3001',
    reuseExistingServer: !process.env.CI,
    timeout: 120000,
  },
]
```

**理由**：
1. Showcase 项目依赖 3001 端口，必须自动启动否则 showcase 测试无法跑
2. Production 模式需要不同的 env，通过 `E2E_PRODUCTION_MODE` 开关切换 3000 端口的模式
3. CI 模式下 `reuseExistingServer: false` 强制重启确保干净环境

**注意**：Next.js dev server 默认占用 3000；通过 `PORT=3001` 启动第二个实例到 3001。需验证 Next.js 是否服从 `PORT` env（标准行为，应服从）。

**选择**：C。

**背景**：`v1.0.0-build-review.md §4.3` 说漏洞在 `/api/data/[...key]/route.ts:37-43`，但本次计划编写时对 `src/app/api` 全量 grep `isSameOrigin|Origin|sameOrigin` **无任何匹配**。

### P0.3 module ID 孤儿清理

| 项 | 说明 |
| --- | --- |
| 文件 | `src/lib/persistence/orphan-cleanup.ts`（新建）、`src/lib/persistence/migration.ts`、`src/lib/persistence/module-library.ts`、`src/components/library/ModuleSwitcher.tsx`（或 library/page.tsx） |
| 现状 | `alc:state:module` 记录 `currentModuleId` 指向不存在 module；根因是 LS 时代 module 序列化不完整，迁移时未校验引用完整性 |

**实施步骤**：

1. **新建 `orphan-cleanup.ts`**：
   ```typescript
   import type { StorageRepository } from './shared/repository'
   import { StorageKeys } from './shared/keys'

   export interface OrphanReport {
     orphanCurrentModuleIds: string[]
     // 可扩展：其他引用型 key 的孤儿
   }

   export function detectOrphans(repo: StorageRepository): OrphanReport {
     const stateRaw = repo.get(StorageKeys.moduleState)  // alc:state:module
     if (!stateRaw) return { orphanCurrentModuleIds: [] }
     const state = JSON.parse(stateRaw) as { currentModuleId?: string }
     if (!state.currentModuleId) return { orphanCurrentModuleIds: [] }

     const moduleRaw = repo.get(StorageKeys.module(state.currentModuleId))
     return moduleRaw
       ? { orphanCurrentModuleIds: [] }
       : { orphanCurrentModuleIds: [state.currentModuleId] }
   }

   export function cleanupOrphans(repo: StorageRepository, report: OrphanReport): void {
     if (report.orphanCurrentModuleIds.length === 0) return
     const stateRaw = repo.get(StorageKeys.moduleState)
     if (!stateRaw) return
     const state = JSON.parse(stateRaw) as { currentModuleId?: string }
     delete state.currentModuleId
     repo.set(StorageKeys.moduleState, JSON.stringify(state))
   }
   ```
   > **注**：实施时需先读 `keys.ts` 确认 `StorageKeys.moduleState` / `module()` 的确切命名。

2. **迁移时校验**（`migration.ts` commit 前）：
   ```typescript
   // 在 commit 阶段，所有 staging 写入 db 后、置 session.status='committed' 前
   const report = detectOrphans(dbRepo)
   if (report.orphanCurrentModuleIds.length > 0) {
     // 记录到迁移日志（warning 级，不阻断）
     logOrphanWarning(report)
     // 选项：自动清理 or 仅记录
     // 推荐：仅记录，让用户在 UI 主动清理（避免迁移期间静默改数据）
   }
   ```

3. **UI 检测**（library/page.tsx 或 ModuleSwitcher 加载时）：

4. **单测**：`orphan-cleanup.test.ts` 覆盖：

   ```typescript
   function resolveHostname(req: NextRequest): string {
     // 优先 X-Forwarded-Host（反代标准 header）
     const xfh = req.headers.get('x-forwarded-host')
     if (xfh) return xfh.split(',')[0].trim()
     // 其次 Forwarded header（RFC 7239）
     const forwarded = req.headers.get('forwarded')
     if (forwarded) {
       const match = /host=([^;,\s]+)/i.exec(forwarded)
       if (match) return match[1].trim()
     }
     // 最后 fallback 到 Host
     return req.headers.get('host') ?? ''
   }

   function assertSameOrigin(req: NextRequest): void {
     const requestHost = resolveHostname(req)
     const origin = req.headers.get('origin')
     if (!origin) return  // 无 Origin 放行（同源 GET / curl）
     const originHost = new URL(origin).host
     if (originHost !== requestHost) {
       throw new Response('Forbidden', { status: 403 })
     }
   }
   ```

3. **单测**：覆盖
   - 直接访问（Host = origin）→ 通过
   - 反代访问（X-Forwarded-Host = origin）→ 通过
   - 跨站（Host ≠ origin）→ 403
   - 无 Origin（curl）→ 通过
