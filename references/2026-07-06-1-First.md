```markdown
**推荐几个开源免费（或高度可自托管）的方案，能把知识转为类似Duolingo的答题闯关形式，同时支持费曼学习法（解释/教学式练习）和知识拆解重组。**

### 1. **最接近的平台级方案：UneeBee（Zoonk）**（虽已归档，但值得参考）
- GitHub: https://github.com/zoonk/uneebee
- 专为创建**Duolingo式交互课程**设计，支持闯关、练习题、进度跟踪等游戏化元素。
- 可用于任意主题（不限语言），强调交互学习（研究显示做中学 retention 高得多）。
- 它是开源的（AGPL），可自托管。虽然项目已归档（2025年），但代码可用，你可以 fork 扩展，或作为模板集成 AI 生成内容。

**适合你的需求**：可以手动/半自动拆解知识成 lessons + questions，支持 progressive difficulty 和练习循环。

### 2. **AI驱动的知识转答题工具（最实用、灵活的“AgentSkill”）**
这些工具能直接从笔记/PDF/文本生成**MCQ、填空、解释题、闪卡**，支持 active recall（主动回忆）和 Feynman-style（解释概念、找gap）：

- **Obsidian Quiz Generator 插件**（强烈推荐）：
  - GitHub: https://github.com/ECuiDev/obsidian-quiz-generator
  - 支持 OpenAI、Gemini、**Ollama（本地LLM，完全免费）** 等模型。
  - 从你的笔记/文件夹生成**交互式考试风格问题**（MCQ、True/False 等）。
  - 完美支持**拆解与重组**：选特定笔记块 → 生成针对性问题 → 练习中解释答案（Feynman）。Obsidian 生态还能结合 mind maps、spaced repetition。

- **其他 GitHub 开源 Quiz Generators**：
  - **QuizCrafter / PDFQuizzer** 等：上传 PDF/笔记 → AI 生成 MCQ 测验，支持即时反馈。
  - **AI-Quiz-Generator**（Next.js + ChatGPT API，可改用本地模型）。
  - 搜索 GitHub “question-generator” 或 “quiz-generator” 有很多（NLP 或 LLM 驱动），很多支持自定义 prompt 来强调 Feynman（e.g. “生成解释题，让用户用简单语言教别人”）。

这些可以作为 **AgentSkill** 集成到你的 AI Agent 流程中（e.g. 用 LangChain / LlamaIndex + Ollama）。

### 3. **语言学习方向的 Duolingo 开源替代（可扩展到通用知识）**
- **LibreLingo**：https://github.com/LibreLingo/LibreLingo —— 完整的开源 Duolingo 克隆，支持课程结构、练习类型。你可以改造成通用主题（添加自定义课程）。
- **LearnHouse**：自托管 LMS，支持 API/CLI/Docker，适合构建完整学习平台。

### 如何实现“费曼 + 拆解重组 + 进步”？
1. **输入知识** → 用 LLM prompt 拆解成原子概念（小块）。
2. **生成题型**：
   - Active Recall：填空/简答/多选。
   - Feynman： “用简单语言解释这个概念，就像教给小学生” + AI 评分 gap。
   - 重组：混合不同主题的问题、变式题、应用场景题。
3. **闯关机制**：用 spaced repetition（Anki 风格或内置）+ 难度递增 +  mastery tracking。
4. **本地免费运行**：Ollama + Obsidian / Streamlit/Gradio 自建 Web App。

**快速上手建议**：
- 先试 Obsidian + Quiz Generator + 本地 Ollama（完全免费，无 API 费）。
- 或用 LangChain 建自定义 Agent：输入文本 → chain 分解 → 生成 questions + feedback loop。
- 如果想全栈 Duolingo 体验，fork LibreLingo 或 LearnHouse，并集成 LLM 生成动态内容。

这些都是开源免费、可自托管的。如果你有特定技术栈（e.g. Python、Next.js）或想集成到现有 Agent 里，我可以帮你细化 prompt 或推荐具体 repo/setup 步骤！
```

~~~markdown
# 我更推荐的 Agent Skill 设计（也是我认为接近多邻国的方法）

建议不要做：

```
知识
↓

生成100题
```

而应该做：

```
知识

↓

Concept Extraction

↓

Skill Graph

↓

Dependency Graph

↓

Difficulty Graph

↓

Learning Units

↓

Question Generator

↓

Adaptive Challenge

↓

Review Planner
```

其中每一个知识点需要拆成：

```
知识点A

↓

Definition

↓

Recognition

↓

Understanding

↓

Application

↓

Comparison

↓

Creation
```

例如：

学习 Prompt Engineering

不是直接问：

> 什么是Few-shot？

而应该变成：

Level1

识别

```
下面哪个属于Few-shot？
```

↓

Level2

理解

```
为什么这个Prompt失败？
```

↓

Level3

应用

```
请修改Prompt。
```

↓

Level4

迁移

```
把Few-shot应用到SQL生成。
```

↓

Level5

费曼

```
向一个高中生解释Few-shot。
```

↓

Level6

组合

Few-shot+Chain of Thought

```

↓

Level7

创新

```
设计一种新的Prompt Pattern。
```
~~~
