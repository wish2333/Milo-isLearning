### 2.3 数据模型实现 — ⚠️ 部分（且属 M2 范围）

**已完成（提前完成的 M2 工作）**：

- `lib/compiler/schemas/` 10 个 Zod Schema，覆盖 Import/Chunk/Concept/Module/Mission/Quiz/Feynman/Feedback/Feynman-Eval 全部 9 个 Agent

- Schema 质量优秀，

  ```
  superRefine
  ```

   

  充分校验跨字段约束：

  - `mission.ts` 校验"前 2 占位符必须 L1 Choice E1"+ Ladder 30-40/30-40/20-30% 分布 + Expression ≥60/≤20/≤20% + expressionLevel 单调非递减
  - `quiz.ts` 校验 options[0]=answer、4 选项长度差 ≤25%、distractors≥3 个 used=true 且不与 answer 重复
  - `feynman.ts` 校验 steps 恰好 6 步且 order 连续、Step 1-4 必须 choice+4 options、Step 5 必须 fill_blank+null
  - `feedback.ts` 校验 score 与 next_action 一致性、score=100 时 gaps 必空、feedback_text 禁用负面词
  - `feynman-eval.ts` 校验 score = Σrubric 得分、gaps = hit=none 的 point 列表

**缺失（属 M1 但未做）**：

- `types/domain.ts`（PRD §8 全部接口：`Module` / `Concept` / `QuizSeries` / `Quiz` / `FeynmanTask` / `FeynmanStep` / `AttemptRecord` / `FeynmanAttempt` / `Mastery` / `ProgressState`）
- LocalStorage key 命名常量（PRD §8 表格、Tech Spec §1.3 `lib/persistence/keys.ts`）

**质量瑕疵**：

- 命名风格不一致：`feedback.ts` 用蛇形 `next_action`（贴近 LLM 输出），PRD §8 `AttemptRecord.nextAction` 用驼峰。**这不是 bug**（Agent IO 与运行时模型本就该分层），但**需要在 M1 完成 `types/domain.ts` 时显式建立映射层**，否则运行时直接吃 Agent 输出会违反 TS strict
